# 备货库存规则核验与实施计划

## 目标

把钉钉文档中的业务规则转成可落库、可校验、可追溯的备货库存语义层。

完成标准：

- 每个业务库存指标有来源表、字段、状态过滤、粒度和刷新频率。
- 涉及积加数据的字段能回到原积加 SCM 知识库一致性映射。
- 默认 0、一对多映射、未匹配映射都有质量规则。
- 业务库存指标可以服务备货计划页面展示和导出。

## 阶段计划

| 阶段 | 目标 | 产出 |
|---|---|---|
| 1. 规则冻结 | 固定钉钉文档中产品库存、FBA、Walmart、TikTok、Shopify、三方仓规则 | 规则评审版 |
| 2. 字段导出 | 导出 SRM、积加、自研 ERP、平台接口、物流跟踪字段 | 字段映射表 |
| 3. 状态核验 | 核验需求池、采购订单、送货单、调拨、货件、物流跟踪状态 | 状态枚举表 |
| 4. 映射核验 | 核验 SKU/MSKU/FNSKU/GTIN/ItemID/国家/仓库归属映射 | 映射质量表 |
| 5. 数据模型 | 建业务库存快照、来源事实表、映射维表和元数据表 | 数据模型草案 |
| 6. 指标验证 | 用样本导出计算指标并和页面口径比对 | 样本校验报告 |
| 7. 看板接入 | 支持业务库存展示、筛选、排序、导出 | BI/页面指标合同 |

## P0 导出清单

| 来源 | 必须导出字段 |
|---|---|
| SRM 需求池 | 需求池单号、物料编码、供应商、数量、库存地点、状态 |
| SRM 采购订单 | PO、物料编码、供应商、数量、库存地点、状态、是否关闭 |
| SRM 送货单 | 送货单号、PO、SKU/物料编码、发货数量、状态、行状态 |
| 积加产品库存 | 仓库、供应链 SKU、可用量、已处理预占、次品量 |
| 积加 FBA 库存 | 仓库、MSKU、FNSKU、本地可售、买家订单预留、运营中心预留、不可售 |
| 积加三方仓库存 | 仓库、三方仓 SKU、供应链 SKU、可用量、预占量、次品量 |
| 调拨单 | 调拨单号、调入仓、调出仓、SKU/MSKU/FNSKU、调拨量、状态 |
| FBA 货件 | 货件号、MSKU、仓库、发货量、货件状态、关联调拨单 |
| 物流跟踪表 | 渠道、国家、目的仓、SKU 明细数量、状态 |
| 仓库归属 | 仓库、渠道、国家、仓库类型 |
| 销售映射 | 供应链 SKU、MSKU、国家、渠道 |
| Walmart 在线商品 | FNSKU、GTIN、ItemID、国家 |

## 字段状态

| 状态 | 用途 |
|---|---|
| `rendered-text-confirmed` | 钉钉页面渲染文本出现 |
| `export-confirmed` | 导出字段已确认 |
| `api-confirmed` | API 字段已确认 |
| `business-default-zero` | 业务规则要求默认为 0 |
| `platform-derived` | 由平台字段计算 |
| `jijia-consistent` | 已与积加 SCM 字段一致性对齐 |
| `mapping-expanded` | 一对多映射导致放大 |
| `mapping-missing` | 未找到映射 |
| `blocked-by-source` | 缺少来源系统导出 |

## 数据质量规则

| 规则 | 检查方式 |
|---|---|
| 默认 0 规则 | Walmart/TikTok/Shopify 等明确默认 0 字段必须写入状态 |
| 状态过滤规则 | 非规则状态不得进入计划库存、采购未交、在途库存 |
| 一对多映射规则 | 记录映射扩展倍数，超过 1 时标记 |
| 未匹配规则 | 缺少 GTIN、MSKU 或销售映射的明细进入未匹配表 |
| 同步时效规则 | 每小时更新来源超过阈值时标记过期 |
| 积加一致性规则 | 积加字段必须在 `jijia-consistency-map` 中找到对应关系 |

## 下一步执行

1. 用本知识库组织业务评审，先确认平台规则是否仍按钉钉文档执行。
2. 按 P0 导出清单获取样本数据。
3. 生成字段映射表和状态枚举表。
4. 用样本数据计算 `STOCK-*` 指标。
5. 和页面展示、导出文件、积加字段字典做三方一致性校验。
