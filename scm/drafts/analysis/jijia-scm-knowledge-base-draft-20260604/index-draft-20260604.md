# 积加 SCM 指标知识库总览

## 结论

本知识库把积加帮助中心中与 `计划-采购-仓库-物流` 四个模块相关的公开文档，整理为供应链指标体系建设的第一版底稿。

当前版本可以作为：

- ERP 页面字段梳理底稿
- 供应链指标口径讨论底稿
- 数据库事实表和维表设计底稿
- 后续从 ERP 导出、API 或数据库取数时的字段核验清单

当前版本不能直接作为：

- 最终数据仓库字段口径
- 已验证的企业内部 ERP 配置事实
- 可上线的指标计算 SQL
- 财务结算或成本核算的唯一依据

原因很明确：积加公开帮助文档解释了业务逻辑、字段含义和部分公式，但很多指标受系统参数、企业实际权限、仓库类型、平台仓报告、物流商接口同步状态影响。最终口径必须用公司实际 ERP 导出、接口返回、页面配置和业务 SOP 复核。

## 四模块边界

| 模块 | 核心业务问题 | 核心数据对象 |
|---|---|---|
| 计划 | 何时补、补多少、用采购还是调拨、何时发货 | 智能补货、红绿线、连续补货建议、采购计划、发货计划、装配计划 |
| 采购 | 计划如何转成供应商订单，交付和付款如何追踪 | 分单任务、采购订单、交货单、供应商产品、供应商对账、供应商仓货权 |
| 仓库 | 实物在哪里、什么状态、如何入库出库、如何追溯 | 产品库存、仓库库存、批次库存、库存流水、入库单、调拨单、调整单 |
| 物流 | 货如何从供应链节点移动，时效和费用如何沉淀 | 头程单、发货单、物流方式、物流渠道、费用看板、估实差异、物流对账 |

## 总体业务流

```text
销售订单/目标销量
  -> 日均消耗量
  -> 目标库存天数
  -> 目标库存量
  -> 推荐采购量/推荐调拨量/推荐发货量
  -> 采购计划/发货计划/装配计划
  -> 分单任务
  -> 采购订单/预装需求/委外订单
  -> 交货单
  -> 发货单/头程单/FBA货件/调拨单
  -> 入库单/平台仓接收/库存流水
  -> 产品库存/仓库库存/批次库存
  -> 物流费用/供应商对账/成本与利润
```

## 指标体系建设原则

1. 先定义业务对象，再定义指标。
2. 先锁定单据生命周期，再计算状态型指标。
3. 库存指标必须区分 `计划量`、`采购量`、`交货量`、`待出运`、`在途`、`入库中`、`良品`、`预占`、`可用`。
4. 补货指标必须记录参数快照，不能只存计算结果。
5. 物流成本必须区分预估、实际、估实差异和跨月成本处理。
6. 供应商对账必须区分采购订单维度、发货单维度、入库单维度。
7. 平台仓库存和 ERP 供应链单据库存是两套来源，不能混为一个事实来源。

## 后续必须核验

- 公司实际启用的智能补货参数。
- FBA 在途使用 `ERP在途` 还是平台报告在途。
- 仓库产品库存中平台仓指标映射配置。
- 产品、MSKU、SKU、平台站点、仓库、供应商之间的主数据映射。
- ERP 导出字段与帮助文档字段是否一致。
- ERP 页面字段、导出字段、API 字段和数据库字段之间的映射状态。
- 积加开放平台或 MCP 是否能提供四模块明细接口。
- 财务对头程成本跨月差异是否采用帮助文档描述的处理方式。
