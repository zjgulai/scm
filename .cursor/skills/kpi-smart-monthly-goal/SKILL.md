# 月度KPI-SMART化与模板落地

## When to Use / 何时使用

- 用户要“写本月KPI/优化KPI模板/改成领导可读版本”。
- 用户明确要求 SMART、可量化、可追踪、可验收。
- 用户要求把 Markdown KPI 填入 Excel 模板。
- 用户提出结构化要求：例如“指标名称不要带日期”“考核标准拆成①②③④”“增加指标类型”。

## When Not to Use / 何时不该使用

- 用户只要做专题分析结论，不需要 KPI 管理模板。
- 用户只要做 PPT Prompt 或汇报故事线，不需要 KPI 指标体系。
- 用户仅需文案润色，不涉及指标口径、目标值、验收机制。

## Core Workflow / 核心流程

1. 识别 KPI 场景与模板结构（通常为业绩/项目/重要工作/加分/减分）。
2. 先对齐项目锚点（专题、里程碑、阶段工作包）再写目标。
3. 将指标改写为“业务名-指标名”，避免空泛名词。
4. 为每条指标补齐：
   - 指标类型（定量/定性）
   - 权重
   - 考核标准（①统计口径/公式 ②③④交付物）
   - 目标值（截止时间 + 数值阈值）
5. 若有 Excel 模板，按列头映射写入并保留模板样式。
6. 最后做一致性检查：权重、类型、公式、口径、可验收性。

## Common Mistakes / 常见错误

- 指标名称写成任务句（如“3月31日前完成…”），而不是指标名。
- 考核标准只写描述，不写统计口径或计算公式。
- 目标值不可量化（如“尽快完成”“高质量完成”）。
- 业绩/项目/重要工作权重不闭合（总和不为100%）。
- KPI 文本与 Excel 模板字段不一致。
