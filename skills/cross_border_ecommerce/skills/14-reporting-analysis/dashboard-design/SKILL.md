# 跨境电商数据看板设计

## 一、Skill概述

数据看板是业务监控的核心工具。好的看板能让用户快速了解业务状态，发现问题并采取行动。本Skill提供从需求分析到设计落地的完整方法论。

**核心价值**：
- 实时监控业务状态
- 快速发现异常问题
- 支持数据驱动决策
- 提升数据使用效率

---

## 二、报表类型与作用

### 2.1 报表分类体系

```
所有报表
├── 明细表（供业务查询）
│   └── 订单明细、用户明细等原始数据
│
├── 统计表（汇总分析）
│   ├── 临时表
│   │   ├── 计算表（业务二次加工）
│   │   └── 取数表（业务直接使用）
│   │
│   └── 固定表
│       ├── 短期表（活动期间支持）
│       └── 长期表（日报、周报、月报）
```

### 2.2 各类报表特点

| 报表类型 | 特点 | 痛点 | 价值点 |
|---------|------|------|--------|
| **临时取数表** | 需求临时、口径不清 | 改来改去、耗费精力 | 明确口径、固定模板 |
| **活动报表** | 周期短、指标集中 | 活动结束即废弃 | 模板复用、沉淀经验 |
| **日常报表** | 周期固定、内容稳定 | 枯燥无聊、没人看 | 自动化、异常预警 |

## 三、看板设计原则

### 3.1 需求梳理五步法

```
步骤1: 明确用户
├── 谁会看这个看板？
├── 他们的角色是什么？
└── 他们关心什么问题？

步骤2: 明确场景
├── 在什么场景下使用？
├── 使用频率是多少？
└── 需要什么级别的数据？

步骤3: 明确目标
├── 看板的目的是什么？
├── 监控？分析？决策？
└── 期望用户做什么动作？

步骤4: 明确指标
├── 核心指标有哪些？
├── 辅助指标有哪些？
└── 对比基准是什么？

步骤5: 明确交互
├── 需要下钻吗？
├── 需要筛选吗？
└── 需要导出吗？
```

### 3.2 指标选择原则

```
指标金字塔
├── 北极星指标（1个）
│   └── 最核心的业务目标
│
├── 一级指标（3-5个）
│   └── 支撑北极星的关键指标
│
├── 二级指标（10-20个）
│   └── 解释一级指标的细节
│
└── 过程指标（按需）
    └── 日常监控的细粒度指标

原则：
- 少即是多，聚焦核心
- 指标可操作、可解释
- 有明确的对比基准
```

### 3.3 布局设计原则

```
标准看板布局
┌─────────────────────────────────────────┐
│  标题 + 时间范围 + 筛选器               │
├─────────────────────────────────────────┤
│  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐       │
│  │KPI 1│ │KPI 2│ │KPI 3│ │KPI 4│ ← 核心指标卡
│  └─────┘ └─────┘ └─────┘ └─────┘       │
├─────────────────────────────────────────┤
│  ┌─────────────────┐ ┌───────────────┐ │
│  │                 │ │               │ │
│  │   主趋势图      │ │   排名/占比   │ │
│  │                 │ │               │ │
│  └─────────────────┘ └───────────────┘ │
├─────────────────────────────────────────┤
│  ┌─────────────────────────────────────┐│
│  │           明细数据表                ││
│  └─────────────────────────────────────┘│
└─────────────────────────────────────────┘

布局原则：
- 重要信息放上方/左侧
- 从概览到细节，层层递进
- 相关指标放在一起
- 留白让界面更清爽
```

## 四、可视化设计

### 4.1 图表选择指南

| 分析目的 | 推荐图表 | 适用场景 |
|---------|---------|---------|
| **看趋势** | 折线图、面积图 | 销售趋势、用户增长 |
| **看对比** | 柱状图、条形图 | 各渠道对比、各产品对比 |
| **看占比** | 饼图、环形图、树图 | 品类占比、渠道占比 |
| **看分布** | 直方图、箱线图 | 价格分布、评分分布 |
| **看关系** | 散点图、气泡图 | 价格与销量、广告与转化 |
| **看排名** | 条形图、排行榜 | 销售排名、商品排名 |
| **看进度** | 仪表盘、子弹图 | 目标完成度 |

### 4.2 配色原则

```
配色建议：
├── 主色调：1-2种品牌色
├── 辅助色：3-5种区分不同类别
├── 警示色：红/黄/绿表示状态
└── 背景色：浅色为主，深色可选

避免：
- 使用过多颜色（不超过7种）
- 相似颜色表示不同含义
- 高对比度刺眼配色
```

### 4.3 交互设计

```
常用交互功能：
├── 筛选器
│   ├── 时间筛选
│   ├── 维度筛选
│   └── 指标筛选
│
├── 下钻
│   ├── 从汇总到明细
│   └── 从高层到低层
│
├── 联动
│   ├── 点击图表联动其他
│   └── 选择维度联动筛选
│
└── 导出
    ├── 导出数据
    └── 导出图片
```

## 六、看板质量评估

### 6.1 评估维度

| 维度 | 评估标准 | 权重 |
|------|---------|------|
| **准确性** | 数据准确、口径一致 | 30% |
| **及时性** | 数据更新及时 | 20% |
| **易用性** | 界面友好、交互流畅 | 20% |
| **有用性** | 指标有价值、能指导决策 | 20% |
| **美观性** | 视觉设计专业 | 10% |

### 6.2 常见问题

| 问题 | 表现 | 解决方案 |
|------|------|---------|
| **没人看** | 访问量低 | 重新梳理需求，聚焦核心 |
| **看不懂** | 用户反馈难理解 | 简化指标，增加说明 |
| **用不上** | 无法指导决策 | 增加对比基准，增加建议 |
| **太慢了** | 加载时间长 | 优化查询，增加缓存 |

## 八、注意事项

1. **从用户出发**：设计前先了解用户需求
2. **少即是多**：聚焦核心指标，避免信息过载
3. **持续迭代**：看板需要根据反馈持续优化
4. **自动化**：能自动化的尽量自动化
5. **文档化**：指标口径、数据来源要文档化
