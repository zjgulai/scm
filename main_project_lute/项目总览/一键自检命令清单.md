# 一键自检命令清单（只读）

> 目的：每次目录调整后，快速确认路径是否一致、关键入口是否可用。  
> 约束：以下命令均为只读检查，不改动文件。

---

## 1）基础结构检查

```bash
cd /Users/pray/project/shopify_analysis
ls -1
```

**期望看到的关键目录**：

- `main_project_lute`
- `skills`
- `knowledge_base`
- `ref`
- `tools`
- `archive`

---

## 2）主项目内聚检查

```bash
cd /Users/pray/project/shopify_analysis
ls -1 main_project_lute
```

**期望包含**：

- `data_example`
- `phase2_mock`
- `phase2_outputs`
- `phase3_mock`
- `phase3_outputs`

---

## 3）旧路径遗留检查（重点）

```bash
cd /Users/pray/project/shopify_analysis
rg -n "cross_border_ecommerce_skills/|mckinsey_ppt_skills/|`data_example/|`phase2_mock/|`phase2_outputs/|`phase3_mock/|`phase3_outputs/" CLAUDE.md AGENTS.md README.md
```

**判定标准**：

- 正常应返回空（无匹配）。
- 若有匹配，表示文档仍有旧路径待修复。

---

## 4）项目级 Skills 路径检查（.cursor）

```bash
cd /Users/pray/project/shopify_analysis
rg -n "`data_example/|cross_border_ecommerce_skills/|mckinsey_ppt_skills/" .cursor/skills
```

**判定标准**：

- 允许出现“旧路径说明文字”（如“不再使用旧路径”）。
- 不应出现仍在实际执行步骤里使用旧路径的命令。

---

## 5）Skills 索引可运行性检查

```bash
cd /Users/pray/project/shopify_analysis
python3 skills/cross_border_ecommerce/skills_index.py "测试搜索"
python3 skills/mckinsey_ppt/skills_index.py "测试搜索"
```

**判定标准**：

- 命令可正常执行并返回搜索结果（或“未找到相关Skills”）。
- 不应报路径不存在或导入失败。

## 7）Git 变更快照（可选）

```bash

git status --short
```

**用途**：

- 复核当前改动范围是否符合预期；
- 在提交前确认没有误改文件。

```bash
cd /Users/pray/project/shopify_analysis && \

ls -1 main_project_lute && \
rg -n "cross_border_ecommerce_skills/|mckinsey_ppt_skills/|`data_example/|`phase2_mock/|`phase2_outputs/|`phase3_mock/|`phase3_outputs/" CLAUDE.md AGENTS.md README.md && \
python3 skills/cross_border_ecommerce/skills_index.py "测试搜索" && \
```
