# Reference: 双引擎知识萃取

## 1) 证据类型分类表

在 Phase 1「论证与证据地图」中，按以下分类标注作者使用的证据：

| 证据类型 | 说明 | 强度评级 |
|---------|------|---------|
| 实证数据 | 统计、实验结果、定量研究 | 高 |
| 系统案例 | 多案例交叉验证、纵向追踪 | 中-高 |
| 单一案例 | 个别故事、轶事 | 低-中 |
| 理论推演 | 基于已有理论的逻辑推导 | 中（取决于前提） |
| 类比论证 | 用其他领域现象类比 | 低（需验证映射） |
| 权威引用 | 引用专家、经典著作 | 低-中（需独立验证） |
| 常识/直觉 | 未提供证据的断言 | 最低 |

标注规则：
- 证据与主张对应关系不清时 → 标注【逻辑跳跃】
- 证据仅支持特定条件 → 标注【适用边界：...】
- 完全缺乏支撑 → 标注【待验证】

## 2) 核心论点链条格式

每条论点按以下结构输出：

```
论点 N：[标题]
├── 问题：作者试图回答什么？
├── 主张：作者的核心观点是什么？
├── 关键逻辑：从问题到主张的推理路径
│   ├── 因果关系：A 导致 B（是否有混淆变量？）
│   ├── 前提假设：该推理依赖哪些未明说的假设？
│   └── 适用边界：在什么条件下成立/不成立？
└── 结论：可带走的判断
```

## 3) Phase Gate 判断标准

Phase 1 完成后，对每个论点/模型做二元判断：

| 标准 | 通过 → 进入 Phase 2 | 不通过 → 搁置 |
|------|---------------------|--------------|
| 逻辑完整性 | 因果链清晰、前提合理 | 逻辑跳跃或循环论证 |
| 证据强度 | 至少中等强度证据 | 仅靠类比或直觉 |
| 可迁移性 | 存在场景独立的方法/框架 | 仅适用于作者特定情境 |
| 实操性 | 可转化为行动 | 纯理念无法落地 |

## 4) 思维模型输出格式

Phase 2 每个模型按以下格式：

```
模型 N：[名称]
├── 一句人话：[用日常语言解释本质]
├── 解决什么：[适用问题类型]
├── 不解决什么：[明确边界]
├── 机会点：
│   ✅ [立即可用的应用 1]
│   ✅ [立即可用的应用 2]
└── 陷阱：
    ⚠️ [常见误用 1]
    ⚠️ [致命陷阱 1]
```

## 5) Action Table 字段规范

Phase 3 行为转化表的每一行字段：

| 字段 | 要求 | 反例 |
|------|------|------|
| 原理名称 | ≤ 8 字，可口语化 | "基于认知负荷理论的..." |
| 一句话本质 | 说明心理/决策机制 | 仅复述书中原话 |
| 最小行动 | ≤ 5 分钟可完成，具体到动作 | "养成好习惯" |
| 典型错误 | 不做该行动时会犯的真实错误 | "效率下降" |

触发条件格式：`当 [具体情境/信号] 出现时 → 执行 [具体动作]`
复盘问题格式：`过去 N 天，[可观测的行为指标] 是否 [预期状态]？`

## 6) 质量门禁详细标准

### Phase 1 质量检查

- 一句话总论是否 ≤ 100 字且无修辞？
- 论点链条是否标注了因果/假设/边界？
- 证据地图是否标注了薄弱环节？
- 反例是否来自不同角度（不是同一个方向的重复质疑）？

### Phase 2 质量检查

- 模型是否用"人话"解释（非学术黑话）？
- 场景推演是否有逐步分析（非结论跳跃）？
- 机会和陷阱是否具体到可操作（非泛泛而谈）？

### Phase 3 质量检查

- 第一性原理是否抽象到机制层（非技巧层）？
- 最小行动是否真的 ≤ 5 分钟？
- 触发条件是否明确到"看到什么信号就做"？
- 复盘问题是否一句话、可回答 yes/no？

### 最终三句判断检查

- 第 1 句是否指向具体价值（非"很有启发"）？
- 第 2 句保留的模型是否在 Phase 2 中验证过？
- 第 3 句是否给出了人群 + 阶段的交叉定位？
