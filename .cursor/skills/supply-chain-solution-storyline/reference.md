# Supply Chain Solution Storyline Reference

## 一、四幕故事线模板

### 第一幕：先把问题从“单点异常”升级为“系统失衡”

标准表达：

- 不是某一项费用高，而是成本、库存、计划与履约同时失衡。
- 如果只按单项压费处理，容易局部改善、全局恶化。

### 第二幕：先升级治理对象，再讨论动作

标准表达：

- 六类费用只能解释“钱花在哪里”。
- 八节点治理才能解释“为什么花、谁负责、怎么改、如何验收”。

推荐八节点：

1. 仓网规划
2. 需求预测
3. 计划排产
4. 头程
5. 仓储
6. 调拨
7. 尾程
8. 逆向

### 第三幕：业务治理与技术平台双引擎联动

业务引擎至少写清：

- 节点 Owner
- 周/月/季节奏
- SOP
- 阈值与升级规则

技术引擎至少写清：

- 指标词典
- 主题宽表
- 看板
- 模型
- 预警
- 建议回写

### 第四幕：项目验收看经营结果，不看系统数量

建议至少保留 3 个结果指标：

- 供应链成本占比
- 全链条周转天数
- 超龄库存占比

同时保留护栏指标，防止“为降本伤经营”。

## 二、管理层标题句模板

### 现状诊断类

- `36%不是单项费用高，而是经营系统已经出现联动失衡`
- `成本、库存与履约体验，本质上是同一条经营链路的问题`

### 治理框架类

- `从六类费用认知，升级为八节点治理，才能把问题变成动作`
- `先抓4个高杠杆节点，才能在最短时间内看到共同改进效果`

### 项目收口类

- `成功标准不是系统上线，而是结果指标改善且没有伤到业务基本盘`
- `如果今天确认5件事，我们就能开始进入联合试点`
