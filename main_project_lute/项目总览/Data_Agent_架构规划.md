## 一、架构核心理念

**从"脚本驱动"到"意图驱动"**

```
传统脚本：用户说"跑分析" → 手动执行脚本 → 输出结果
Data Agent：用户说"帮我分析毛利率" → Agent理解 → 自动调用Skills → 输出结果
```

## 五、核心原则

1. **不破坏现有文件** - 新代码放在新目录
2. **复用现有Skills** - 通过调用而非重写
3. **每步测试** - 确保可用性
4. **用户目标** - "用户说一句话 → 自动完成分析+PPT"

## 六、使用流程（目标）

```python
# 用户输入
user_input = "帮我分析独立站毛利率下降的原因"

# Agent处理
agent = DataAgent()
result = agent.execute(user_input)

# 自动执行
# 1. 意图识别 → 毛利率归因分析
# 2. Skills匹配 → margin-attribution (相关性98%)
# 3. Skill执行 → 按方法论分析
# 4. PPT生成 → 按Skill模板输出

# 返回结果
# {
#   "status": "success",
#   "ppt_file": "outputs/margin_attribution_2026.pptx",
#   "summary": "独立站毛利率下降3.4pp，主要原因...",
#   "insights": [...]
# }
```
