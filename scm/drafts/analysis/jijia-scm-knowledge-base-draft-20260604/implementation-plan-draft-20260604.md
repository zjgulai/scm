# 积加 SCM 指标体系建设计划

## 目标

建设覆盖 `计划-采购-仓库-物流` 的供应链指标体系和数据库基础，使业务团队能回答以下问题：

- 哪些产品会缺货，为什么缺货。
- 哪些计划需要采购、调拨、发货或装配。
- 采购计划转采购订单是否及时。
- 供应商是否按期、按量、按质交付。
- 货物现在处于计划、采购、交货、发货、在途、入库还是平台仓接收阶段。
- 库存是良品、次品、预占、冻结、共享还是锁站点。
- 物流时效、费用、估实差异如何影响补货和利润。

## 阶段计划

### 阶段 1：文档基线和业务对象抽取

**目标**：把公开帮助文档转为可分析的业务对象清单。

任务：

- 固定四模块文档清单。
- 抽取单据、状态、字段、公式、参数。
- 标记文档中图片才展示、文本未展开的字段。
- 形成文档级证据清单。

完成标准：

- 每个模块都有业务对象表。
- 每个指标都有来源链接。
- 不把未验证字段写成最终事实。

### 阶段 2：ERP 实际字段核验

**目标**：确认公开文档中的字段在公司 ERP 中是否真实可见、可导出、可接口化。

任务：

- 从积加下载中心或页面导出四模块核心列表。
- 记录导出文件名、导出时间、导出人、筛选条件。
- 对照帮助文档字段，建立 `doc_field -> export_field -> api_field` 映射。
- 记录缺失字段、重命名字段、权限隐藏字段。

完成标准：

- 每个核心事实表至少有一个真实数据来源。
- 每个字段有 `confirmed`、`missing`、`permission-hidden`、`parameter-dependent` 状态。
- 仓库模块先按 `erp-field-validation-checklist-draft-20260604.md` 的 P0/P1 队列执行导出和字段映射。

### 阶段 3：指标口径定稿

**目标**：把业务字段转成公司指标字典。

任务：

- 按业务域定义指标：补货、采购执行、供应商交付、库存健康、物流时效、物流成本。
- 为每个指标定义粒度、筛选条件、状态口径、时间口径。
- 对参数驱动指标建立参数快照。
- 对平台仓库存和 ERP 单据库存建立双来源标识。

完成标准：

- 指标字典中每个指标有唯一编码。
- 指标口径包含数据来源、粒度、计算逻辑、业务负责人。
- 指标血缘能追溯到源单据或源报告。

### 阶段 4：数据库底座建设

**目标**：建立能承载供应链指标的数据模型。

任务：

- 建设主数据维表：产品、MSKU、仓库、平台站点、供应商、物流方式、物流渠道、日期。
- 建设单据事实表：采购计划、采购订单、交货单、发货单、头程单、入库单、调拨单。
- 建设库存快照表：产品库存、仓库库存、平台仓库存、批次库存。
- 建设库存流水表：每次良品/次品数量变化。
- 建设成本事实表：物流预估费用、实际费用、估实差异、供应商对账。
- 建设指标血缘表和参数快照表。

完成标准：

- 每张事实表有明确粒度。
- 每张表有业务主键和技术主键。
- 同一指标能追溯到来源表和来源字段。

### 阶段 5：数据质量规则

**目标**：在指标上线前识别断链、重复、延迟和异常。

任务：

- 校验单据链路完整性：采购计划 -> 采购订单 -> 交货单 -> 发货单 -> 入库单。
- 校验库存守恒：入库、出库、调拨、调整与库存流水一致。
- 校验状态合法性：已作废单据不进入有效库存或有效采购量。
- 校验成本差异：预估费用、实际费用、差异率超阈值标记。
- 校验时效：采购交期、物流时效、入库处理天数与实际日期偏差。

完成标准：

- 每个核心指标至少有一条质量规则。
- 异常能定位到单据号、SKU、仓库、供应商或物流商。

### 阶段 6：BI 和运营知识库

**目标**：把指标体系转成供应链运营可用的分析入口。

任务：

- 计划看板：缺货风险、推荐采购、推荐调拨、推荐发货。
- 采购看板：计划分单率、订单交付、供应商交期、采购入库。
- 仓库看板：库存结构、良品可用、预占冻结、库龄、库存流水异常。
- 物流看板：在途、到仓、时效偏差、费用估实差异、对账异常。
- 知识库：指标定义、血缘图、常见问题、口径变更记录。

完成标准：

- 看板指标和知识库指标使用同一指标编码。
- 口径变更能追踪历史版本。
- 业务用户能从指标跳转到定义和血缘说明。

## 关键风险

| 风险 | 影响 | 处理方式 |
|---|---|---|
| 帮助文档字段与公司导出字段不同 | 指标无法落库 | 建立字段映射表，不直接套文档字段名 |
| 智能补货公式受参数控制 | 同一指标不同公司结果不同 | 必须采集参数快照 |
| 平台仓库存来自平台报告，ERP 单据来自供应链流程 | 库存口径混淆 | 建立 `source_system` 和 `inventory_source_type` |
| 物流费用存在跨月估实差异 | 利润口径偏差 | 单独建估实差异事实表 |
| 供应商仓货权依赖参数和实物类型 | 请款口径错误 | 供应商货权单独建模，不合并进普通采购入库 |
| 图片中字段未能文本化 | 字段遗漏 | 后续用页面截图或导出补齐 |
