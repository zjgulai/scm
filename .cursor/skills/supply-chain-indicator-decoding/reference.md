# Supply Chain Indicator Decoding Reference

## 一、北极星定义模板

推荐起手式：

```text
北极星指标 = 结果总量 / 经营规模
```

本轮样例：

```text
供应链成本占比 = 供应链总成本 / 销售额
```

设计提示：

- 先确认分子范围是否统一。
- 再确认分母口径是否唯一。
- 最后再做层级拆解，不要边拆边改口径。

---

## 二、L0-L3 解码模板

### L0

- 北极星指标

### L1

- 战略层分类，要求 MECE
- 一般控制在 6-10 类，便于管理层理解

### L2

- 每个 L1 下的策略抓手
- 一般对应具体治理主题或能力模块

### L3

- 一线执行指标
- 必须能进入 SOP、周会、看板或预警阈值

---

## 三、指标字典字段模板

| 字段 | 说明 |
|---|---|
| 指标名称 | 建议具体到动作对象 |
| 所属层级 | L1/L2/L3 |
| 公式/口径 | 可直接计算 |
| 数据源 | ERP / WMS / TMS / OMS / BI |
| 更新频率 | 日 / 周 / 月 / 季 |
| Owner | 业务 Owner |
| 建议阈值 | 预警线 / 健康区间 |
| 上层映射 | 映射哪个 L2/L1/北极星 |

---

## 四、约束对设计模板

| 约束对 | 左侧指标 | 右侧指标 | 健康区间 | 熔断规则 | 管理含义 |
|---|---|---|---|---|---|
| 库存 vs 断货 | 周转天数、超龄占比 | 断货率、断货损失 | 周转与断货同时达标 | 断货恶化时暂停压库存 | 不允许为好看周转牺牲可售性 |

写法要求：

- 必须同时给出健康区间与熔断动作。
- 约束对不能只描述冲突，要给出管理规则。

---

## 五、节点映射建议

优先映射到 8 节点：

1. 仓网规划
2. 需求预测
3. 计划排产
4. 分仓头程配送
5. 仓储
6. 调拨
7. 履约尾程
8. 补发/退货/返仓

同时建议给出：

- 业务 Owner
- 数据 Owner
- 系统 Owner
- 对应算法模型
