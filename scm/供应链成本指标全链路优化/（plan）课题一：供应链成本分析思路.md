## 本地重建说明

本节为基于当前项目本地资料重建的分析方案，不等同于钉钉原文复制。当前重建依据来自 Data 层 5 个文件、`ref/books/供应链36%方案_Page6节点/供应链课题/供应链成本分析正式稿_费用深拆版.md` 和 `供应链成本分析正式稿_叙事版.md`。

## 1. 分析目标

本分析不只回答“供应链成本率高不高”，而是回答四个管理问题：

| 问题 | 判断口径 | 输出 |
|---|---|---|
| 成本改善是否真实 | 总成本率、销售增速、成本增速、四项费用 YoY | 经营结论 |
| 改善是否被对冲 | 采购费率 vs 物流三项费率 | 对冲测算 |
| 高成本来自哪里 | 区域/渠道/GTM/负责人/国家月份 | 责任归因 |
| 下一步怎么治理 | 节点指标、Owner、动作、验收 | 执行清单 |

## 2. 标准分析链路

```text
口径锁定
  -> 总盘趋势
  -> 四项费用拆解
  -> 区域/渠道/GTM 交叉归因
  -> 国家月份异常识别
  -> 负责人责任矩阵
  -> 八节点动作映射
  -> 管理层结论与执行优先级
```

## 3. 输入数据

| 数据对象 | 必要字段 | 来源 |
|---|---|---|
| 成本宽表 | 销售额、采购成本、头程成本、尾程成本、仓储成本、总成本率 | `dwt_supply_chain_cost` |
| 库存宽表 | 在库、在途、未交 PO、库存金额、周转天数、库龄 | `dwt_inventory_health` |
| 供应商宽表 | 供应商、采购金额、准交率、质检、降本率 | `dwt_supplier_performance` |
| 履约宽表 | 订单、包裹、国家、渠道、时效、配送费、异常 | `dwt_fulfillment_stability` |
| 逆向宽表 | 退货单、原因、补发、返仓、报废、处理成本 | `dwt_reverse_logistics` |

## 4. 分析模块

### 4.1 总盘趋势

| 指标 | 计算 | 解释 |
|---|---|---|
| 供应链总成本率 | 供应链总成本 / 销售额 | 结果层北极星 |
| 销售增速 | 本期销售额 / 上期销售额 - 1 | 分母增长质量 |
| 成本增速 | 本期供应链成本 / 上期供应链成本 - 1 | 成本扩张速度 |
| 成本率变动 | 本期成本率 - 上期成本率 | 总体效率变化 |

### 4.2 四项费用拆解

| 费用项 | 判断重点 | 典型结论 |
|---|---|---|
| 采购 | 是否吞噬物流降本 | 采购上行是下一阶段 P0 战场 |
| 头程 | 是否由急采、装载、渠道方式驱动 | 头程要按渠道和航线治理 |
| 尾程 | 是否由国家、渠道、承运商驱动 | 尾程需进入成本+体验双 KPI |
| 仓储 | 是否由库龄、周转、逆向恢复驱动 | 仓储不是仓租问题，是库存效率问题 |

### 4.3 交叉归因

| 视角 | 目的 | 输出 |
|---|---|---|
| 区域 | 定位北美/欧洲/拉美等区域结构差 | 区域费用瀑布 |
| 渠道 | 区分 Amazon、offline、Shopify 等成本指纹 | 渠道驱动项表 |
| GTM | 识别品线增长质量 | GTM 增长质量分级 |
| 负责人 | 绑定经营责任 | owner 责任矩阵 |
| 国家月份 | 识别异常放大类型 | z-score 异常清单 |

## 5. 诊断规则

| 场景 | 判定 | 优先行动 |
|---|---|---|
| 表观改善、结构承压 | 总成本率下降，但采购费率上升抵消物流改善 | 采购端拆 SKU/供应商/品线 |
| 区域复合高成本 | 区域成本率高，采购和尾程均高 | 区域责任制 + 履约路径复盘 |
| 渠道成本指纹差异 | 不同渠道第一驱动项不同 | 按渠道设差异化 KPI |
| 销售下滑被动放大 | 成本金额下降慢于销售额 | 控固定费用、调库存和仓网 |
| 成本绝对值上升驱动 | 销售增长但成本增长更快 | 查节点费用、急采、承运商、仓储 |
| 混合驱动 | 销售与成本均波动且成本率抬升 | 联合复盘销售节奏和履约动作 |

## 6. 输出物

1. `供应链成本效率整合洞察及方案V1.0`：管理层结论与行动方案。
2. `供应链指标体系与分析视图类型清单`：指标到视图的映射。
3. `供应链多维分析星图看板设计`：看板信息架构。
4. `kp01-kp04` 执行方案：重点节点动作、输入、输出和验收。

## 7. 分析护栏

1. 不把四项费用重算口径等同于全链路口径；退换补发和直邮缺失时必须说明。
2. 不用单一总成本率判断治理成败，必须同时看采购、头程、仓储、尾程结构。
3. 不把低成本等同于好结果，必须联动断货、时效、履约事故和退货。
4. 不在指标字典未确认前做跨系统强结论。
