# 跨境电商指标异常检测 Skill

## 概述

异常检测是业务监控的核心环节。本Skill提供系统化的异常检测方法，帮助用户：

- **识别异常波动**: 区分正常波动与异常波动
- **分类波动类型**: 一次性/周期性/持续性
- **选择合适算法**: 根据指标特性选择最佳检测方法
- **建立多窗口基线**: 短期/中期/长期基线并行

### 支持的7个核心指标

| 指标 | 波动类型 | 首选基线与窗口 | 检测顺序 |
|------|---------|---------------|---------|
| 销售额(GMV) | 周期性+一次性突发 | 日值；7日周窗口；去年同期 | ①日环比±30%→P1；②7周平滑>2σ→P1；③超预测区间+退款/折扣异常→P0 |
| 营业收入 | 与GMV同步，受退款/折扣影响 | 日/周；月累计对比 | ①日环比±25%、月同比>15%→P1；②3σ自适应；③下降且毛利为负→P0 |
| 毛利额 | 持续性趋势+一次性异常 | 日/周；滚动30天 | ①毛利额<0→P0；②环比降>20%或30日Z<-2→P0/P1 |
| 毛利率 | 稳定基线+对折扣/广告敏感 | 日/周；30天滑动 | ①毛利率<10%→P0；②30日滑动相对下降>2σ→P0/P1；③折扣率↑且毛利率↓→审价复核 |
| 广告费率 | 波动受投放节奏影响 | 日；7日移动平均 | ①7日平均上升>30%且ROAS降>20%→P2；②单日极端上升>50%→P1 |
| 退款率 | 一次性+持续性（质量/物流） | 日/周；14天窗口 | ①日退款率>5%→P1；②周同比上升>50%或S-ESD异常→P1；③单SKU/单承运商集中且金额大→P0 |
| 折扣率 | 促销期大幅波动+临期清仓 | 日/活动期；历史促销周期平滑 | ①活动白名单内放宽阈值；②非活动期上升>15pp→P2；③毛利率显著下降→升级P1/P0 |

## 核心功能

### 1. 三类波动识别

**一次性波动（短期突发）**:
- 偶发的、突然性的波动
- 原因：投流、大促、节日、KOL推广
- 检测方法：绝对值阈值、日环比

**周期性波动（周/季/年循环）**:
- 与时间节点强相关
- 以周、季、年为循环节点
- 检测方法：同比、周期平滑

**持续性波动（长期趋势性变化）**:
- 从某时间开始持续上涨/下降
- 原因更深刻：订单结构变化、环境因素
- 检测方法：移动平均、ARIMA趋势

### 2. 绝对值预警

**适用场景**: 一次性波动，有明确业务底线

**方法**: 设定固定阈值，超出即预警

**阈值设定优先级**:
1. **战略指引**: 公司OGSM战略拆解的目标值
2. **历史最佳表现**: 历史正常情况下的波动范围
3. **同行业平均水平**: 对标行业基准

**示例**:
```
退货率指标不得高于8%
销售额达成率不得低于85%
毛利率不得低于10%
毛利额不得小于0
```

### 3. 相对值预警

#### 3.1 同比环比

**同比(YoY)**: 与去年同期比较
```
YoY = (本期值 - 去年同期值) / 去年同期值 × 100%
```

**环比(MoM)**: 与上一期比较
```
MoM = (本期值 - 上期值) / 上期值 × 100%
```

**适用场景**:
- 有明显周期性的指标，同比更合理
- 短期波动监控，环比更敏感

#### 3.2 周期平滑

**方法**: 利用多个时间点的周期数据进行平滑

**比较值公式**:
```
b = Σ(K_ij × X_ij)

其中:
- K_ij 为平滑系数，总和为1
- X_ij 为历史同周期数据
```

**优点**: 避免单点异常数据的误导

#### 3.3 3σ原则（自适应阈值）

**理论基础**: 假设检验和大数定律

**公式**:
```
Z = (X - μ) / σ

异常判定: |Z| > z_α

其中:
- μ = 历史数据均值
- σ = 历史数据标准差
- z_α = 置信水平对应的Z值
```

**常用置信水平**:
| 置信水平 | z值 | 含义 |
|---------|-----|------|
| 90% | 1.65 | 90次在正常范围 |
| 95% | 1.96 | 95次在正常范围 |
| 99% | 2.58 | 99次在正常范围 |

**示例**:
```
历史30天销售额均值: $100,000
历史30天销售额标准差: $15,000
今日销售额: $145,000

Z = (145,000 - 100,000) / 15,000 = 3.0

|Z| > 2.58 (99%置信水平) → 判定为异常
```

### 4. 时间序列模型

#### 4.1 移动平均法

**简单移动平均(SMA)**:
```
SMA_n = (X₁ + X₂ + ... + Xₙ) / n
```

**指数移动平均(EMA)**:
```
EMA_t = α × X_t + (1-α) × EMA_{t-1}

其中 α = 2/(n+1)
```

**特点**: EMA对近期数据赋予更高权重，反应更灵敏

#### 4.2 ARIMA模型

**全称**: 自回归移动平均模型 (AutoRegressive Integrated Moving Average)

**原理**: 利用过去几个数据点预测下一个数据点，确定预测区间

**检测逻辑**:
```
1. 拟合ARIMA模型
2. 生成下一期预测值和置信区间
3. 如果实际值超出置信区间 → 判定为异常
```

**适用场景**: 持续性趋势分析、预测区间检测

#### 4.3 S-ESD算法

**全称**: Seasonal Extreme Studentized Deviate

**原理**: 结合季节性分解和ESD异常检测

**步骤**:
```
1. STL分解时间序列（趋势+季节+残差）
2. 对残差应用ESD异常检测
3. 识别统计显著的异常点
```

**适用场景**: 有季节性的指标异常检测（如退款率）

### 5. 多时间窗口基线

**三层窗口并行**:

| 窗口类型 | 时间跨度 | 用途 |
|---------|---------|------|
| 短期窗口 | 7日 | 识别瞬时波动 |
| 中期窗口 | 30日 | 识别周期性偏离 |
| 长期窗口 | 90日/同周期历史 | 识别趋势性变化 |

**双轨判定**:
```
目标达成轨: Attain = 实际值 / 拆解目标
基线偏离轨: Z分数或分位数Bias

两轨并行触发不同等级告警
```

## 使用方法

### 输入数据要求

**时间序列数据格式**:
```csv
date,metric_value,target_value
2024-01-01,100000,95000
2024-01-02,105000,95000
2024-01-03,98000,95000
...
```

**字段说明**:
| 字段 | 说明 | 格式 |
|------|------|------|
| date | 日期 | YYYY-MM-DD |
| metric_value | 指标值 | 数值 |
| target_value | 目标值 | 数值（可选） |

### 分析流程

```
步骤1: 数据加载与预处理
- 读取时间序列数据
- 检查数据完整性
- 处理缺失值

步骤2: 确定波动类型
- 分析指标特性
- 判断波动类型（一次性/周期性/持续性）

步骤3: 选择检测方法
- 一次性波动 → 绝对值阈值、环比
- 周期性波动 → 同比、周期平滑
- 持续性波动 → 移动平均、ARIMA

步骤4: 计算基线与偏差
- 计算多窗口基线
- 计算Z分数/分位数偏差
- 计算目标达成率

步骤5: 异常判定
- 应用阈值判定
- 生成异常等级

步骤6: 输出报告
- 异常检测结果
- 可能原因分析
- 建议排查步骤
```

## 算法详解

### S-ESD算法完整示例

**场景**: 检测退款率异常

**步骤**:
```
1. STL分解:
   原始序列 = 趋势项 + 季节性项 + 残差项

2. 对残差应用ESD检验:
   - 计算残差的中位数(M)和MAD
   - 计算每个点的ESD统计量
   - 与临界值比较

3. 确定异常点数量:
   - 使用最大归一化残差检验
   - 确定统计显著的异常点

4. 输出异常点位置和程度
```

**优点**: 能检测出季节性数据中的真正异常，避免季节性波动被误判为异常

### ARIMA预测区间

**参数说明**:
```
ARIMA(p, d, q):
- p: 自回归阶数
- d: 差分次数
- q: 移动平均阶数
```

**预测区间计算**:
```
预测值: ŷ_t
预测标准误: σ_t
95%置信区间: [ŷ_t - 1.96×σ_t, ŷ_t + 1.96×σ_t]

实际值超出区间 → 判定为异常
```

## 示例场景

### 示例1: 销售额异常检测

**场景**: 2024-03-15销售额为$75,000，判断是否异常

**数据**:
| 日期 | 销售额 |
|------|-------|
| 2024-03-08 | $110,000 |
| 2024-03-09 | $108,000 |
| 2024-03-10 | $105,000 |
| 2024-03-11 | $112,000 |
| 2024-03-12 | $109,000 |
| 2024-03-13 | $106,000 |
| 2024-03-14 | $111,000 |
| 2024-03-15 | $75,000 |

**检测**:
```
7日均值: $108,714
7日标准差: $2,460
Z分数: (75,000 - 108,714) / 2,460 = -13.7

|Z| > 2.58 → 严重异常
日环比: (75,000 - 111,000) / 111,000 = -32.4% < -30% → 异常
```

**结论**: P1级异常，需立即排查

### 示例2: 退款率周期性异常检测

**场景**: 检测退款率是否异常

**历史同期数据**:
| 周次 | 退款率 |
|------|-------|
| 4周前 | 4.2% |
| 3周前 | 4.5% |
| 2周前 | 4.3% |
| 1周前 | 4.6% |
| 本周 | 7.8% |

**检测**:
```
周期平滑基线: (4.2% + 4.5% + 4.3% + 4.6%) / 4 = 4.4%
本周偏差: 7.8% - 4.4% = 3.4pp

阈值: 日退款率>5% → P1

7.8% > 5% → 异常
```

**结论**: P1级异常，需排查退款原因

## 注意事项

### 方法选择

| 场景 | 推荐方法 | 原因 |
|------|---------|------|
| 有明确底线 | 绝对值阈值 | 直接有效 |
| 周期性指标 | 同比/周期平滑 | 消除季节性 |
| 持续性趋势 | 移动平均/ARIMA | 识别趋势变化 |
| 高精度需求 | S-ESD | 考虑季节性分解 |

### 常见问题

**Q: 阈值如何设定？**
A: 优先级：战略指引 > 历史最佳表现 > 行业平均

**Q: 如何避免误报？**
A: 使用多窗口基线、活动日历白名单、联动规则

**Q: Z分数选择多少合适？**
A: 一般选择1.96（95%置信）或2.58（99%置信）

### 活动感知

维护活动日历和白名单，活动期内使用放宽阈值：

| 活动类型 | 阈值调整策略 |
|---------|------------|
| 大促活动 | GMV阈值放宽至±50% |
| 常规促销 | GMV阈值放宽至±40% |
| 品推活动 | 广告费率阈值上浮20% |
| 清仓活动 | 毛利率阈值下调5pp |
