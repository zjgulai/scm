# 跨境电商数据清洗 Skill

## 概述

本Skill专为跨境电商数据分析场景设计，提供全面的数据清洗能力。跨境电商数据通常来自多个渠道（平台后台、ERP、广告系统、物流系统），存在格式不统一、缺失、异常等问题，本Skill帮助用户快速标准化数据，为后续分析奠定基础。

## 核心功能

### 1. 数据质量评估
- **完整性检查**: 统计每列缺失值比例
- **唯一性检查**: 检测重复记录
- **有效性检查**: 识别异常值、离群点
- **一致性检查**: 格式统一性、逻辑一致性

### 2. 缺失值处理
- **删除策略**: 删除缺失严重的行/列（阈值可配置）
- **填充策略**:
  - 数值型: 均值、中位数、众数填充
  - 分类型: 众数填充或"未知"标签
  - 时间序列: 前向填充、后向填充、插值
- **预测填充**: 使用机器学习预测缺失值（高级选项）

### 3. 异常值处理
- **检测方法**:
  - IQR法（四分位距）: 适合正态分布
  - Z-Score法: 适合对称分布
  - 孤立森林: 适合高维数据
  - 业务规则: 基于领域知识设定阈值
- **处理策略**:
  - 标记: 添加异常标记列，保留原始值
  - 修正: 用边界值或预测值替换
  - 删除: 移除异常记录

### 4. 数据标准化
- **格式统一**:
  - 日期格式: 统一为YYYY-MM-DD
  - 货币格式: 统一数值类型，记录币种
  - 文本格式: 去除空格、统一大小写
- **编码转换**: 自动检测并统一字符编码
- **单位换算**: 支持重量、尺寸等单位统一

### 5. 重复数据处理
- **完全重复**: 删除完全相同的记录
- **部分重复**: 根据关键字段（订单号+SKU）去重
- **模糊重复**: 使用相似度算法识别近似重复

**支持格式**:
- CSV (.csv)
- Excel (.xlsx, .xls)
- JSON (.json)
- Parquet (.parquet)

| 产品信息 | sku, product_name, category | 产品标识和分类 |

### 清洗流程

```
步骤1: 数据加载与初步检查
- 读取文件，检测编码
- 统计行列数、数据类型
- 生成数据概览报告

步骤2: 数据质量诊断
- 缺失值统计与可视化
- 异常值检测与标记
- 重复数据识别
- 格式问题检测

步骤3: 清洗策略制定
- 根据数据特征推荐清洗方案
- 用户确认或调整策略
- 设定各字段清洗规则

步骤4: 执行清洗
- 按策略处理缺失值
- 处理异常值
- 去除重复数据
- 标准化格式

步骤5: 质量验证
- 对比清洗前后指标
- 生成数据质量评分
- 输出清洗日志
```

## 算法说明

### 缺失值填充策略

**数值型字段**:
```python
if 缺失率 < 5%:
    使用中位数填充（抗异常值）
elif 缺失率 < 30%:
    使用KNN预测填充
else:
    标记为单独类别或删除该列
```

**分类型字段**:
```python
if 缺失率 < 10%:
    使用众数填充
else:
    填充为"未知"类别
```

### 异常值检测算法

**IQR法**:
```
Q1 = 第25百分位数
Q3 = 第75百分位数
IQR = Q3 - Q1

异常值范围: < (Q1 - 1.5×IQR) 或 > (Q3 + 1.5×IQR)
极端异常: < (Q1 - 3×IQR) 或 > (Q3 + 3×IQR)
```

**Z-Score法**:
```
Z = (X - μ) / σ
|Z| > 3 视为异常值
```

## 示例

### 示例1: 销售数据清洗

**输入数据** (sales_raw.csv):
```csv
order_id,order_date,sku,quantity,price,country
1001,2024/01/15,A001,2,29.99,US
1002,2024-01-16,A002,-1,15.50,uk
1003,2024/01/17,A001,3,,US
1004,2024/01/18,A003,1,9999.00,DE
1005,2024/01/15,A001,2,29.99,US
```

**检测到的问题**:
1. 日期格式不统一 (2024/01/15 vs 2024-01-16)
2. quantity为负数 (-1)
3. price缺失值
4. price异常值 (9999.00)
5. 国家代码大小写不统一 (US vs uk)
6. 完全重复记录 (1001和1005)

**清洗策略**:
1. 日期统一为YYYY-MM-DD格式
2. 负数quantity标记为异常
3. price缺失用同类SKU中位数填充
4. price>1000标记为异常（保留供人工复核）
5. 国家代码统一为大写
6. 删除完全重复记录

**输出**:
- 清洗后数据: sales_cleaned.csv
- 质量报告: data_quality_report.md
- 清洗日志: cleaning_log.txt

## 注意事项

### 数据安全
- 清洗过程在本地执行，数据不会上传
- 敏感字段（客户姓名、地址）可配置脱敏

### 性能优化
- 大数据集(>100万行)建议分块处理
- 可使用采样快速预览清洗效果

### 常见问题

**Q: 清洗后数据量变少很多，正常吗？**
A: 如果原始数据质量较差（大量缺失、重复），清洗后数据量减少是正常的。建议检查清洗日志，确认删除原因。

**Q: 异常值被删除了，如何恢复？**
A: 清洗时选择"标记"策略而非"删除"，异常值会被标记但保留在数据中，便于后续人工复核。

**Q: 能否保存清洗规则供下次使用？**
A: 可以。清洗完成后会生成规则配置文件(cleaning_rules.json)，下次可直接加载应用。

## 高级功能

### 自定义清洗规则

通过JSON配置文件定义清洗规则:
```json
{
  "rules": [
    {
      "column": "price",
      "type": "numeric",
      "missing_strategy": "median",
      "outlier_method": "iqr",
      "outlier_action": "flag"
    },
    {
      "column": "country",
      "type": "categorical",
      "missing_strategy": "mode",
      "standardize": "uppercase"
    }
  ]
}
```

### 批量处理

支持同时处理多个文件:
```python
# 批量清洗某目录下所有CSV文件
process_directory("/data/sales/", pattern="*.csv", rules=rules)
```
