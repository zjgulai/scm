# 目录架构总览与资产分类

## 1. 定位

`main_project_lute/` 是当前仓库的主项目核心，承载 Momcozy 母婴跨境电商数据分析、四大专题故事线、数据契约、Data Agent 原型、专题分析脚本、mock 数据与阶段汇报产物。

根目录下的 `skills/`、`ref/`、`knowledge_base/`、`scm/`、`tools/` 是服务主项目的支撑层，不应与 `main_project_lute/` 混放，也不应继续把业务总结、临时输出、实验结果直接堆在根目录。

---

## 2. 分类原则

| 分类 | 判定标准 | 主要落点 |
|---|---|---|
| 主项目正式资产 | 当前仍服务四大专题、Data Agent 或管理层汇报闭环 | `main_project_lute/` |
| 主项目阶段交付 | Phase2/Phase3 已生成且用于复盘、验收或演示的产物 | `main_project_lute/phase*_outputs/` |
| 主项目测试夹具 | 支撑阶段管道运行的 mock 数据 | `main_project_lute/phase*_mock/` |
| 主项目实验资产 | 仍在验证中的脚本、输入输出与阶段管道 | `main_project_lute/data_example/实验管道/`、`main_project_lute/data_example/scripts/experimental/` |
| 通用能力资产 | 可复用 Skills、PPT 方法、发布包 | `skills/`、`_publish_claude_skills/` |
| 支撑知识资料 | 外部书籍、行业报告、指标方法论、角色画像 | `ref/`、`knowledge_base/` |
| 供应链专题资产 | 供应链成本、KPI 树、指标体系、执行方案 | `scm/` |
| 归档材料 | 历史输出、旧总结、日志、旧对话、失活但有参考价值的材料 | `archive/` |
| 临时产物 | 运行输出、截图、调试文件、可丢弃中间结果 | `tmp/` |

## 5. 支撑层分类结果

### 5.1 `ref/`

`ref/` 是业务参考资料库，包含书籍萃取、行业报告、数据指标字典、角色画像和 Phase2 输入参考。它服务主项目分析，但不是主项目执行产物。

建议继续按以下语义维护：

| 子目录 | 分类 |
|---|---|
| `ref/books/` | 跨境电商、品牌出海、供应链、数据分析书籍萃取 |
| `ref/company_info/` | Momcozy/母婴出海行业资料 |
| `ref/data_index/` | 存量 BI 指标字典 |
| `ref/data_report/` | 外部行业报告 |
| `ref/roles_skills/`、`ref/roles_skills_info/` | Agent 角色画像与技能映射 |
| `ref/phase2_io/` | Phase2 输入参考 |
| `ref/aippt/` | PPT 参考资料 |

### 5.2 `knowledge_base/`

`knowledge_base/` 是长期方法论资料库，体量较大，包含大量 PDF/PPT/XLSX。当前不建议并入 `main_project_lute/`，否则会污染主项目工作区。

| 子目录 | 分类 |
|---|---|
| `knowledge_base/data_ability/` | 数据分析、指标体系、经营分析、看板、用户分析等方法论 |
| `knowledge_base/reason_analysis/` | 归因与原因分析资料 |
| `knowledge_base/insights/` | 洞察沉淀 |

### 5.3 `scm/`

`scm/` 是供应链成本专题库，不是通用知识库。它与主项目的 Phase3/供应链分析有关，但当前更像一个可独立阅读的专题包。

建议后续二选一：

| 方案 | 适用条件 |
|---|---|
| 保留 `scm/` 顶层专题库 | 供应链专题会持续独立演进 |
| 迁入 `main_project_lute/供应链专题/` | 供应链专题明确成为主项目四大专题的一部分 |
