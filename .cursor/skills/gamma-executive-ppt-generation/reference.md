# Gamma Executive PPT Reference

## 一、总 Prompt 必备字段

每次生成前，至少明确以下内容：

- 汇报主题
- 汇报对象
- 目标页数
- 汇报目标
- 叙事顺序
- 风格要求
- 统一禁止项
- 必须使用的数据

---

## 二、推荐 Prompt 结构

### 1. 总指令

写清：

- 这是一套几页中文管理层汇报
- 面向谁
- 风格是什么
- 不允许做什么

### 2. Deck Narrative

按页序写明逻辑链，常用格式：

1. 为什么现在必须改
2. 为什么这不是单部门问题
3. 我们共同优化什么
4. 为什么升级治理框架
5. 哪些约束不能被破坏
6. 为什么先抓这些试点
7. 业务与 IT 如何共创
8. 如何分阶段落地
9. 如何衡量成功
10. 今天请拍板什么

### 3. 逐页 Prompt

每页建议至少写 4 项：

- 标题
- 目的
- 必须包含
- 版式/设计要求

### 4. 关键页补丁 Prompt

用于修正最关键、最容易跑偏的页面。

---

## 三、生成后检查清单

首轮生成完成后，优先检查：

1. 页数是否正确
2. 标题是否都是结论句
3. 是否擅自加了新数据
4. 是否出现追责语气
5. 是否把一页拆成多个小页
6. 是否还有后续人工编辑空间

---

## 四、推荐人工微调顺序

1. 先修标题
2. 再修 5 个关键页
3. 再统一视觉风格
4. 最后补备注区/演讲稿

这样效率通常最高。
