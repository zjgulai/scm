# 跨境电商多指标联动分析 Skill

## 概述

单一指标异常往往不能反映全貌，多指标联动分析能更精准识别问题。本Skill提供系统化的多指标联动方法，帮助用户：

- **建立联动规则**: 定义指标间的关联关系
- **精准判断异常**: 通过多指标组合降低误报
- **自动升级告警**: 联动异常时自动提升告警等级
- **快速定位根因**: 通过指标关系图谱追踪问题

### 核心指标关系图谱

```

                    │                        GMV (销售额)                          │
                    │          = 访客数 × 转化率 × 客单价                           │
                    └───────────────────────────┬─────────────────────────────────┘
                                │
                ┌───────────────┼───────────────┐
                ▼               ▼               ▼
        ┌───────────┐   ┌───────────┐   ┌───────────┐
        │  广告费率  │   │  折扣率   │   │  退款率   │
        │ (获客成本) │   │ (促销成本) │   │ (售后成本)│
        └─────┬─────┘   └─────┬─────┘   └─────┬─────┘
              │               │               │

                              │
```

## 核心功能

### 1. 五大核心联动规则

#### 规则1: GMV↓ 且 退款率↑

**联动条件**:
```
日环比GMV下降 > 20% 且 退款率上升 > 30%
```

**判定逻辑**:
- GMV下降可能是正常波动
- 但同时退款率上升 → 说明存在严重问题
- 可能原因：质量问题、物流问题、描述不符

**告警等级调整**: P1 → P0

**自动动作**:
```
1. 自动带出退款样本（最近50条）
2. 生成退款原因分布报告
3. 触发退款排查工单
4. 分配给客服团队
```

#### 规则2: 毛利率↓ 且 折扣率↑

**联动条件**:
```
毛利率下降 > 2σ 且 折扣率上升 > 10pp
```

**判定逻辑**:
- 毛利率下降可能由多种原因
- 但同时折扣率上升 → 说明促销侵蚀利润
- 需要审视促销策略

**告警等级调整**: P2 → P1

**自动动作**:
```
1. 触发审价/促销复核
2. 建议暂停相关优惠
3. 生成折扣效率报告
4. 通知运营负责人
```

#### 规则3: 营业收入↓ 且 毛利额<0

**联动条件**:
```
营业收入下降 > 15% 且 毛利额为负
```

**判定逻辑**:
- 营业收入下降本身需要关注
- 但同时毛利为负 → 严重亏损
- 需要立即干预

**告警等级调整**: P1 → P0

**自动动作**:
```
1. 告警升级P0
2. 带出退款/折扣占比
3. 带出成本构成明细
4. 通知财务+运营负责人
```

#### 规则4: 广告费率↑ 且 ROAS↓

**联动条件**:
```
广告费率上升 > 30% 且 ROAS下降 > 20%
```

**判定逻辑**:
- 广告费率上升可能是投放策略调整
- 但同时ROAS下降 → 投放效率恶化
- 需要优化投放

**告警等级调整**: 维持P2（但需关注）

**自动动作**:
```
1. 联合判定投放效率问题
2. 带出Campaign级表现
3. 带出预算消耗明细
4. 通知投放负责人
```

#### 规则5: 折扣率异常(非活动期) 且 毛利率显著↓

**联动条件**:
```
非活动期折扣率异常 且 毛利率下降 > 5pp
```

**判定逻辑**:
- 非活动期折扣率异常可能是规则错误
- 同时毛利率显著下降 → 利润严重受损
- 需要立即干预

**告警等级调整**: P2 → P1/P0

**自动动作**:
```
1. 升级P1/P0
2. 触发促销复核与冻结
3. 检查折扣叠加情况
4. 通知运营+财务负责人
```

### 3. 联动检测执行逻辑

```
Step 1: 单指标检测 → 识别异常指标
Step 2: 联动规则匹配 → 检查是否满足联动条件
Step 3: 告警升级/合并 → 调整告警等级或合并告警
Step 4: 自动动作执行 → 触发对应的自动化动作
Step 5: 通知增强 → 在告警内容中标注联动关系
```

### 4. 指标关系图谱

#### 成本流向

```
采购成本 → 头程物流 → 关税 → FBA费用 → 平台佣金 → 退款损耗 → 广告成本 → 折扣成本
    │                                                       │
    └───────────────────────────────────────────────────────┘
                              │
                              ▼
                        实际净利润
```

#### 影响关系

| 指标 | 正向影响 | 负向影响 |
|------|---------|---------|
| GMV | ↑访客数、↑转化率、↑客单价 | ↓流量、↓转化、缺货 |
| 营业收入 | ↑GMV | ↑退款、↑折扣、↑平台费 |
| 毛利额 | ↑销量、↑售价、↓成本 | ↑采购成本、↑物流费、↑退款 |
| 毛利率 | ↑售价、↓成本 | ↑折扣率、↑广告费率 |
| 广告费率 | - | ↑CPC、↓ROAS |
| 退款率 | - | 质量问题、物流问题、描述不符 |
| 折扣率 | - | 促销活动、折扣叠加 |

## 使用方法

### 配置联动规则

```yaml
correlation_rules:
  - id: "RULE-001"
    name: "GMV下降+退款率上升"
    description: "GMV下降且退款率上升，可能存在质量或物流问题"

    conditions:
      - metric: "gmv"
        type: "mom"
        operator: "<"
        threshold: -0.20
      - metric: "refund_rate"
        type: "change"
        operator: ">"
        threshold: 0.30

    actions:
      - type: "upgrade_alert"
        from: "P1"
        to: "P0"
      - type: "pull_data"
        dataset: "refund_samples"
        limit: 50
      - type: "create_ticket"
        template: "refund_investigation"
        assignee: "customer_service"
```

### 联动检测流程

```
1. 持续监控各指标
2. 检测到单指标异常
3. 匹配联动规则
4. 检查其他指标是否满足条件
5. 满足条件 → 触发联动告警
6. 执行自动动作
7. 发送增强通知
```

## 注意事项

### 联动规则设计原则

1. **相关性**: 只有真正相关的指标才建立联动
2. **方向性**: 明确指标间的影响方向
3. **阈值合理**: 避免过于敏感或迟钝
4. **可解释**: 联动逻辑要有业务含义

### 常见问题

**Q: 联动规则越多越好吗？**
A: 不是。过多规则会增加复杂度和误报，建议控制在10个以内。

**Q: 如何判断两个指标是否应该联动？**
A: 分析历史数据的相关性，相关性>0.5且业务逻辑支持才建立联动。

**Q: 联动告警误报怎么办？**
A: 调整阈值、增加时间窗口条件、增加更多联动指标。

### 最佳实践

1. **从核心规则开始**: 先实现5大核心联动规则
2. **持续优化**: 定期评估联动规则准确率
3. **记录案例**: 沉淀典型案例供后续参考
4. **业务参与**: 联动规则需要业务方确认
