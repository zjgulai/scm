# 跨境电商转化漏斗分析 Skill

## 概述

本Skill帮助跨境电商卖家分析用户在购买路径上的转化情况。通过构建转化漏斗、计算各环节转化率、识别流失点，提供科学的转化率优化建议，提升整体销售效率。

## 核心功能

### 1. 转化漏斗构建
- **标准电商漏斗**: 曝光 → 点击 → 浏览 → 加购 → 结算 → 支付
- **自定义漏斗**: 根据业务特点定义环节
- **多维度漏斗**: 按渠道、设备、地区等维度细分

### 2. 转化率计算
- **环节转化率**: 当前环节完成数 / 上一环节进入数
- **整体转化率**: 最终完成数 / 初始进入数
- **相对转化率**: 与基准（如行业平均）对比

### 3. 流失分析
- **流失率**: 1 - 转化率
- **流失量**: 上一环节人数 - 当前环节人数
- **流失价值**: 流失量 × 平均客单价（估算）

### 4. 归因分析
- **流失原因**: 页面加载慢、价格高、运费贵等
- **用户分群**: 新老客户、不同来源的转化差异
- **A/B测试**: 验证优化效果

### 分析流程

```
步骤1: 数据准备
- 加载漏斗数据
- 数据验证（确保数据合理性）
- 按维度分组

步骤2: 计算转化率
- 计算各环节转化率
- 计算整体转化率
- 计算流失率

步骤3: 识别流失点
- 找出转化率最低的环节
- 计算流失量和流失价值
- 分析流失原因

步骤4: 对比分析
- 同比/环比趋势
- 多维度对比
- 行业基准对比

步骤5: 优化建议
- 针对流失环节提出建议
- 优先级排序
- 预期效果估算
```

## 核心公式

### 环节转化率
```
CVR_i = 完成数_i / 进入数_i

示例:
- 加购数: 7,000
- 浏览数: 20,000
- 加购转化率 = 7000 / 20000 = 35%
```

### 整体转化率
```
整体CVR = 最终完成数 / 初始进入数

示例:
- 曝光: 1,000,000
- 支付: 1,200
- 整体CVR = 1200 / 1000000 = 0.12%
```

### 流失率
```
流失率_i = 1 - CVR_i

流失量_i = 进入数_i - 完成数_i

流失价值 = 流失量 × 平均客单价（估算）
```

### 预期提升价值
```
预期增收 = 流失价值 × 预期转化率提升

示例:
- 结算流失价值: $135,000
- 预期转化率提升: 33%
- 预期增收 = 135000 × 33% = $44,550
```

## 示例

### 示例1: 诊断支付环节流失

**场景**: 结算→支付转化率仅28.6%，需要诊断原因

**分析步骤**:
1. 分析用户行为：在结算页停留时长、跳出点
2. 调研用户反馈：为什么不完成支付
3. 技术排查：支付成功率、错误日志

**诊断结果**:
| 原因 | 占比 | 说明 |
|------|------|------|
| 运费太贵 | 35% | 运费占商品价格20%+ |
| 支付失败 | 20% | 信用卡拒收率高 |
| 支付方式少 | 15% | 缺少PayPal、本地支付 |
| 页面加载慢 | 10% | 移动端加载超过5秒 |
| 其他 | 20% | 各种零散原因 |

**优化方案**:
- 满额包邮策略降低运费敏感度
- 接入更多支付方式
- 优化移动端性能

### 示例2: A/B测试验证优化效果

**场景**: 测试简化结账流程的效果

**测试设计**:
- 对照组：5步结账（原流程）
- 实验组：3步结账（优化流程）
- 流量分配：50% / 50%
- 测试周期：2周

**测试结果**:
| 指标 | 对照组 | 实验组 | 提升 |
|------|--------|--------|------|
| 结算数 | 2,100 | 2,150 | +2.4% |
| 支付数 | 600 | 720 | +20% |
| 支付转化率 | 28.6% | 33.5% | +17% |

**结论**: 简化结账流程显著提升支付转化率，建议全量上线

## 注意事项

### 数据准确性
- 确保各环节数据口径一致
- 排除爬虫、机器流量
- 处理跨设备、跨会话情况

### 漏斗定义
- 漏斗环节要符合业务逻辑
- 环节不宜过多（5-7个为宜）
- 允许非线性路径（如直接购买）

### 时间窗口
- 设定合理的转化归因窗口
- 不同产品购买周期不同
- 考虑跨天转化情况
