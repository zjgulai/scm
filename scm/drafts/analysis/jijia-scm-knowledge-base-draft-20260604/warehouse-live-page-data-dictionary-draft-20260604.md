# 积加 ERP 仓库模块实采页面数据字典

## 采集范围

本轮数据来自 Codex 自带浏览器登录后的积加 ERP 仓库导航页面，采集方式为只读读取页面可见字段、筛选器、页签、按钮和表格字段。未点击新增、提交、确认、导入、调整、下单、发货等会改变业务状态的动作。

临时原始提取文件：

- `tmp/outputs/jijia-warehouse-live-extraction-20260604.json`
- 二次重采证据摘要：`drafts/analysis/jijia-scm-knowledge-base-draft-20260604/warehouse-live-rerun-evidence-draft-20260604.md`

## 二次重采修正

首次采集中，部分页面因为加载时间过短、默认筛选未稳定，被误判为无数据。2026-06-04 二次重采后，40 个仓库子页中有 24 个页面在当前筛选下可见表格数据，1 个页面有非零状态页签但列表被筛选压空，12 个页面当前筛选为空，3 个页面属于扫码、发货、组包类操作入口而非列表报表页。

后续建库时必须区分 4 种页面证据状态：

| 状态 | 处理方式 |
|---|---|
| 有表格数据 | 可进入页面字段字典、实体识别和事实表候选 |
| 状态有量/表格受筛选 | 保留状态口径，另行校验二级筛选和导出字段 |
| 当前筛选为空 | 保留字段结构，不得推断业务无历史数据 |
| 操作页/无列表 | 进入作业流程知识库，不作为列表事实表来源 |

采集覆盖 7 个分组、40 个子页面：

| 分组 | 页面数 | 业务域 |
|---|---:|---|
| 库存 | 8 | 现存量、平台仓、批次、库龄、流水 |
| 入库管理 | 2 | 收货、入库、质检 |
| 库内管理 | 9 | 调拨、调整、盘点、其他出入库、状态变更、装配、装箱 |
| 出库管理 | 10 | 销售出库、拣货、验货发货、物流下单、包裹、运输、B2B、全托管 |
| 三方仓管理 | 4 | 三方仓入库预报、三方仓入库、三方仓销售出库、大货出库 |
| 库存稽核 | 3 | 三方仓差异、FBA 月报差异、FBA 库存对账 |
| 基础配置 | 4 | 仓库、库位、容器、批次初始化 |

## 数据层分类

仓库模块的数据应拆成 5 类表，不建议把页面字段直接合并成一张宽表：

| 数据层 | 说明 | 代表页面 |
|---|---|---|
| 主数据 | 仓库、库位、容器、SKU、平台站点、供应商、物流商 | 仓库资料、库位资料、容器资料 |
| 库存快照 | 某一时间点的数量、货值、库龄和状态 | 仓库库存、产品库存、自营仓库存、平台仓库存、库龄分析 |
| 库存流水 | 每一次库存数量或状态变化 | 库存流水、批次流水、调整单、其他出入库、状态变更单 |
| 作业单据 | 入库、质检、调拨、装配、装箱、拣货、发货、物流 | 入库单、质检单、调拨单、装配单、销售出库单、拣货任务、包裹单 |
| 对账稽核 | ERP 与三方仓、FBA 报告、财务库存对账差异 | 三方仓库存差异、月度FBA报告差异、FBA库存对账 |

## 采集限制

- 本轮二次重采已确认多数核心页面存在数据；仍有部分页面受当前登录用户、时间、仓库、状态、质量状态等筛选影响显示为空。
- `入库单` 有 `待入库:818` 的状态页签证据，但当前列表仍为空，需要用导出或 API 校验二级筛选。
- `库龄分析` 在当前自带浏览器会话中仍为空；用户本地 Chrome 截图曾显示数据，后续需要按正确仓库/平台筛选单独复采。
- `三方仓销售出库单` 表格出现记录样本但分页总数显示 0，需导出校验分页口径。
- 本轮没有执行导出，因此没有拿到隐藏列、后端字段名、API 字段名和字段类型。
- `仓库资料` 页面存在真实仓库主数据，本草稿只沉淀字段，不复制仓库明细值。
- 库龄货值按页面说明使用本位币，汇率取系统汇率设置中当前月份官方汇率。
