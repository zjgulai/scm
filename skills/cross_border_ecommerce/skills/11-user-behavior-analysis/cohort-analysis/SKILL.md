# 跨境电商群组/留存分析 Skill

## 概述

本Skill通过同期群分析(Cohort Analysis)帮助跨境电商卖家理解用户留存和生命周期价值。通过跟踪不同时期获取的用户群体，分析其留存率、复购行为和LTV，为用户运营策略提供数据支持。

## 核心功能

### 1. 同期群留存分析
- **群组定义**: 按首次购买月份划分用户群组
- **留存率计算**: N月后仍有购买行为的用户比例
- **留存曲线**: 可视化各群组的留存衰减

### 2. 群组对比分析
- **横向对比**: 同一月龄不同群组的留存差异
- **纵向对比**: 同一群组不同月龄的留存变化
- **趋势分析**: 群组留存率的改善趋势

### 3. LTV计算
- **简单LTV**: ARPU × 平均生命周期
- **累积LTV**: 各月贡献累加
- **预测LTV**: 基于历史数据预测

### 4. 用户生命周期分析
- **活跃周期**: 从首次购买到最后一次购买的平均时长
- **购买间隔**: 平均复购周期
- **生命周期阶段**: 新手、成长、成熟、衰退、流失

### 分析流程

```
步骤1: 数据准备
- 加载交易数据
- 识别每个客户的首次购买日期
- 按首次购买月份划分群组

步骤2: 计算留存
- 对每个群组，计算每月的活跃用户数
- 计算留存率 = 活跃用户 / 群组初始用户

步骤3: 构建留存表
- 行：群组（按月）
- 列：月龄（0, 1, 2, ...）
- 值：留存率或活跃用户数

步骤4: 计算LTV
- 计算每个群组的累积消费金额
- 除以群组初始用户数得到人均LTV

步骤5: 生成洞察
- 识别留存趋势
- 对比群组表现
- 提出运营建议
```

## 核心公式

### 留存率
```
留存率_n = 第n月活跃用户数 / 首月用户数

示例:
- 群组初始用户: 1,000
- 第3月活跃用户: 150
- 第3月留存率 = 150 / 1000 = 15%
```

### 流失率
```
流失率_n = 1 - 留存率_n
或
流失率_n = (上期活跃 - 本期活跃) / 上期活跃

示例:
- 第1月留存: 32%
- 第1月流失率 = 1 - 32% = 68%
```

### LTV（累积法）
```
LTV_n = Σ(M_i) / 初始用户数

其中:
- M_i = 第i月该群组的总消费金额

示例:
- 初始用户: 1,000
- 0月消费: $45,000
- 1月消费: $11,000
- 2月消费: $8,000
- LTV_2 = (45000 + 11000 + 8000) / 1000 = $64
```

### LTV（简单法）
```
LTV = ARPU × 平均生命周期

其中:
- ARPU = 总收入 / 总用户数 / 月数
- 平均生命周期 = 1 / 流失率（月）

示例:
- 月ARPU: $45
- 月流失率: 10%
- 平均生命周期 = 1 / 0.1 = 10个月
- LTV = 45 × 10 = $450
```

### 复购率
```
复购率 = 有复购的用户数 / 总购买用户数

示例:
- 总购买用户: 10,000
- 有2次以上购买: 3,500
- 复购率 = 3500 / 10000 = 35%
```

## 注意事项

### 群组定义
- 按首次购买月划分最常用
- 也可按获客渠道、设备类型等划分
- 群组大小要足够（建议>100）

### 时间窗口
- 新群组数据不完整是正常的
- 长期留存需要更长时间观察
- 预测模型可补充不完整数据

### 留存定义
- 严格定义：有购买行为
- 宽松定义：有访问/互动
- 根据业务选择合适定义

### 预测留存模型

```python
# 使用历史数据预测未来留存
def predict_retention(historical_retention, months_to_predict):
    """
    基于历史留存曲线预测未来留存
    """
    from scipy.optimize import curve_fit

    def retention_curve(t, a, b):
        # 幂律衰减模型
        return a * (t + 1) ** (-b)

    # 拟合历史数据
    x = np.arange(len(historical_retention))
    params, _ = curve_fit(retention_curve, x, historical_retention)

    # 预测未来
    future_x = np.arange(len(historical_retention),
                         len(historical_retention) + months_to_predict)
    predicted = retention_curve(future_x, *params)

    return predicted
```
