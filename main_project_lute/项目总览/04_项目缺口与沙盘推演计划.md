## 一、逻辑总览与缺口摘要

- **设计基线**：01 矩阵 + 05 表结构 定义「要什么数」；Phase1 故事线 + Agent 画像 定义「谁用、谁产、产什么」；交叉线编排 定义「跨专题数据流」。
- **当前缺口**：部分计划未落盘、外部数据未细化到可采集级、Agent/Skill 偏通用、多条分析链缺实现或未闭环；需用**虚拟数据**先做模块 I/O 与沙盘推演，再接真数。

## 三、沙盘推演与验收

- **Mock 生成**：执行 `python3 data_example/scripts/generate_phase3_sandbox_mock.py`，在 `phase3_mock/` 下生成订单、退款、订单行、VOC 汇总/趋势/品牌、渠道月/流量/健康度、营销日/ROI 等 mock 表。
- **管道消费约定**：各 Phase3 管道优先读真实中间结果，若无则回退读 `phase3_mock/` 下同名 mock（与 Phase2 交叉线3 mock 回退一致）。
- **评估报告**：每次沙盘跑批后填写《沙盘评估报告》，模板与保存位置见 [沙盘推演计划](../Phase3_全专题与运营化/沙盘推演计划.md) 第五节；建议路径：`Phase3_全专题与运营化/沙盘评估报告_YYYYMMDD.md`。
