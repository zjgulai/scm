# KPI Smart Skill Reference

## 1) 本轮对话沉淀的核心改进点（总结+补充）

### A. 从“写任务”升级到“写指标”

- 错误示例：`3月31日前完成专题②...`
- 正确示例：`订单质量提升-核心归因交付达成率`

原则：**标题只写指标，不写日期**；日期放在“目标值”中。

### B. SMART 落地方式

- **S（具体）**：指标必须绑定业务对象（订单、VOC、数据就绪、联调、汇报页）。
- **M（可量化）**：必须有计算公式或计数口径。
- **A（可达成）**：目标值与当前阶段匹配（如本月v1、>=2轮验证）。
- **R（相关）**：直接映射四大专题与 Phase 工作包。
- **T（有时限）**：统一放在目标值中（如“截至3月31日”）。

### C. 考核标准统一结构

每条指标必须写成：

- ① 统计口径/计算公式
- ② 交付物A（主产物）
- ③ 交付物B（补充产物）
- ④ 交付物C（评审/引用/闭环证明，可选但建议保留）

### D. 指标类型判定规则

- **定量指标**：满足“可计算公式 + 数值目标值”。
- **定性指标**：主要为“是否完成/是否采纳/是否触发”的二值判断。

判定建议：
- 业绩/项目/重要工作优先写定量；
- 加分/减分通常写定性（必要时也可定量）。

## 2) 输出模板（推荐）

```md
### n）指标名称：业务名-指标名
- 指标类型：定量指标/定性指标
- 权重：x%
- 考核标准：
  ① 统计口径=...
  ② 交付物A：...
  ③ 交付物B：...
  ④ 交付物C：...
- 目标值：截至YYYY-MM-DD，...
```

## 3) Excel 模板落地规范

若模板列头为：`目标项 | 指标类型 | 指标名称 | 权重 | 考核标准 | 目标值`

- 一行一个指标；
- “考核标准”单元格内保留 ①②③④；
- 开启换行，避免内容溢出；
- 不改模板结构（工作表名、列顺序、公式单元格）。

## 4) 质量检查清单（交付前）

- [ ] 指标名称不含日期
- [ ] 每条都有“指标类型”
- [ ] 每条都有公式/统计口径
- [ ] 每条考核标准均为①②③④结构
- [ ] 一～三项权重总和=100%
- [ ] 目标值包含时间与阈值
- [ ] 与四大专题/里程碑有明确映射
- [ ] 若写入Excel，列头映射正确且换行可读
