# 积加物流模块业务逻辑与指标血缘

## 模块定位

物流模块负责回答：

1. 货如何从调出仓移动到目的仓。
2. 物流方式和渠道是什么。
3. 预计何时到仓，实际何时到仓。
4. 预估费用、实际费用和差异如何影响成本。

物流模块既影响计划模块的补货时效，也影响仓库模块的在途和入库，还影响财务利润中的头程成本。

## 核心子模块

| 子模块 | 业务作用 | 关键对象 |
|---|---|---|
| 物流方式 | 维护头程运输方式、时效、计费方式 | 物流方式、时效、体积重系数 |
| 物流渠道 | 维护物流商可用渠道 | 三方仓物流、API 物流、平台物流、自定义物流、平台仓配送 |
| 头程单 | 桥接交货单/调拨单与 FBA 货件 | 头程单、来源单、FBA货件、差异量 |
| 发货单 | 记录出运、到仓和完成流程 | 发货量、确认出运、预计到仓、实际到仓 |
| 头程物流下单 | 与物流商接口交互 | 下单、揽收、费用、轨迹 |
| 运费试算 | 发货前比较渠道成本 | 运费模板、物流渠道、预估费用 |
| 物流费用看板 | 汇总发货单和包裹费用 | 预估费用、实际费用 |
| 估实差异看板 | 管理预估和实际费用差异 | 估实差异金额、差异总金额、已出库成本 |
| 头程物流对账 | 按发货单/物流商/费用项对账 | 差异率、差异金额、处理意见 |

## 业务逻辑

### 物流方式

物流方式维护运输类型、计费方式、体积重系数和运输时效。它直接参与：

- 目标库存天数。
- 最迟发货日期。
- 发货计划预计入库日期。
- 调拨单体积重。
- 发货计划总重量和总体积重。

关键字段：

- 物流方式。
- 状态。
- 发货国家。
- 计费方式。
- 体积重系数。
- 物流时效。

### 物流渠道

物流渠道分为：

- 三方仓物流。
- API 对接物流。
- 平台物流。
- 自定义物流。
- 平台仓配送。

渠道决定是否能自动下单、是否能同步费用和轨迹、是否只能用于特定平台订单。

### 头程单

头程单用于 FBA 头程管理，充当交货单或调拨单与 FBA 货件之间的桥梁。它的关键价值是把 ERP 单据数量和平台货件申报数量关联起来。

关键字段：

- 头程单号。
- 目的仓库。
- 亚马逊站点。
- 来源单号。
- 物流方式。
- FBA 货件。
- 货件申报量。
- 单据量。
- 差异量。

### 发货单

发货单是物流执行的核心事实表。不同目的仓的生成方式不同：

- 发往本地仓的发货单可由交货单发货后生成。
- 发往海外仓的发货单可能需要手动创建并关联单据。
- 发往 FBA 仓的发货单通常在 FBA 货件流程中创建。

关键日期：

- 出运日期。
- 预计入库日期。
- 实际发货日期。
- 实际到仓日期。
- 开始收货日期。

关键状态：

- 草稿。
- 提货中。
- 已出运。
- 已完成。
- 已作废或已发作废。

### 头程物流下单

头程物流下单用于和物流商接口交互。它能支持下单、取消、打印箱唛、查看揽收、费用和轨迹。

同步逻辑需要单独记录：

- 物流商订单信息拉取频率。
- 揽收信息拉取频率。
- 费用信息拉取频率。
- 轨迹信息拉取频率。
- 拉取范围。

### 物流费用与估实差异

物流费用存在预估和实际两套数据。费用对利润有强影响，不能简单以实际费用覆盖预估费用。

关键逻辑：

- 发货时录入预估物流费用，预估成本可能先参与利润。
- 后续录入实际费用后，未出库库存可更新实际成本。
- 已销售出库且跨月的部分，可能继续沿用预估成本。
- 估实差异需要独立补差或调减。

### 头程物流对账

头程物流对账按发货单、物流商和费用项生成或更新对账单。它支持差异率和最大差异金额阈值，用于识别异常对账。

## 指标字典

| 指标编码 | 指标 | 粒度 | 口径 |
|---|---|---|---|
| SCM-LOG-001 | 物流时效天数 | 物流方式 + 调出仓 + 调入仓 | 维护的预计运输耗时 |
| SCM-LOG-002 | 体积重 | 发货单/调拨单 + SKU | 体积按物流方式体积重系数换算 |
| SCM-LOG-003 | 货件申报量 | 头程单 + 货件 + SKU | FBA 货件中的申报数量 |
| SCM-LOG-004 | 单据量 | 头程单 + 来源单 + SKU | 绑定交货单或调拨单中的产品数量 |
| SCM-LOG-005 | 头程差异量 | 头程单 + SKU | 货件申报量与单据量的差异 |
| SCM-LOG-006 | 发货量 | 发货单 + SKU | 发货单产品发货数量 |
| SCM-LOG-007 | 实际发货日期 | 发货单 | 发货单确认出运日期 |
| SCM-LOG-008 | 实际到仓日期 | 发货单 | 亚马逊签收或货代确认到仓日期 |
| SCM-LOG-009 | 开始收货日期 | 发货单 + SKU | 平台或自营仓开始接收的日期 |
| SCM-LOG-010 | 预计物流费用 | 发货单 + 物流商 + 费用项 | 发货时录入或试算的费用 |
| SCM-LOG-011 | 实际物流费用 | 发货单 + 物流商 + 费用项 | 物流商或人工录入的实际费用 |
| SCM-LOG-012 | 估实差异金额 | 发货单 + 费用项 | 实际费用 - 预估费用 |
| SCM-LOG-013 | 估实差异率 | 发货单 + 费用项 | 估实差异金额 / 预估费用 |
| SCM-LOG-014 | 差异总金额 | 发货单 + SKU/费用项 | 估实差异金额 × 已出库量 |
| SCM-LOG-015 | 对账异常标记 | 对账单 | 差异率或差异金额超过阈值 |

## 指标血缘

```text
物流方式/物流渠道/运费模板
  -> 发货计划/调拨单/发货单
  -> 头程单/FBA货件/物流商下单
  -> 费用录入/费用同步
  -> 物流费用看板
  -> 估实差异看板
  -> 头程物流对账
  -> 成本和利润
```

上游数据：

- 物流方式。
- 物流渠道。
- 运费模板。
- 交货单。
- 调拨单。
- FBA 货件。
- 物流商接口。

下游数据：

- 发货单。
- 入库单。
- 平台仓接收。
- 物流费用。
- 头程物流对账。
- 利润和成本分析。

## 数据建模建议

| 表 | 粒度 | 必备字段 |
|---|---|---|
| `dim_logistics_method` | 物流方式 + 目的仓 | 计费方式、体积重系数、时效、状态 |
| `dim_logistics_channel` | 物流渠道 | 物流商、渠道类型、是否 API 对接、适用场景 |
| `fact_first_mile_order_line` | 头程单 + 货件 + SKU | 来源单、货件申报量、单据量、差异量 |
| `fact_shipment_order_line` | 发货单 + SKU | 发货量、预计入库、实际发货、实际到仓、开始收货 |
| `fact_logistics_order_sync` | 物流商订单 + 同步批次 | 订单状态、揽收、轨迹、费用同步时间 |
| `fact_logistics_cost_line` | 发货单 + 物流商 + 费用项 | 预估费用、实际费用、差异金额、差异率 |
| `fact_logistics_reconciliation_line` | 对账单 + 费用项 | 对账状态、差异处理意见、异常标记 |

## 后续核验清单

- 公司使用哪些物流方式、渠道和运费模板。
- 物流方式时效是否按目的仓分别维护。
- 头程物流商接口是否启用，费用和轨迹是否自动同步。
- 发货单费用是物流录入、财务录入还是物流商同步。
- 估实差异是否纳入财务补差流程。
- 物流费用是否按 SKU、箱、重量、体积或单据维度分摊。
