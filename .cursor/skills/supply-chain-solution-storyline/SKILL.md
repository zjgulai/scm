# 供应链方案故事线与评估

## When to Use / 何时使用

- 用户要把诊断结果讲成管理层可拍板故事线。
- 用户要做方案评估报告、立项叙事、联合汇报。
- 用户强调“怎么讲给总监听”“怎么收口到拍板事项”。

## When Not to Use / 何时不该使用

- 只做指标树和执行指标设计。
- 只做 Gamma 输出。
- 只重算某个分析脚本结果。

## Core Workflow / 核心流程

1. 定义北极星、当前基线、目标区间。
2. 将问题从单项异常升级为经营链路失衡。
3. 用四幕叙事组织：现状 -> 框架 -> 联动 -> 验收。
4. 收口到试点范围、路线图、拍板事项。

## Common Mistakes / 常见错误

- 只讲费用项，不讲经营链路。
- 只讲诊断，不讲 Owner 与节奏。
- 语气偏追责，导致协同失败。
- 只讲系统功能，不讲经营动作与护栏。
