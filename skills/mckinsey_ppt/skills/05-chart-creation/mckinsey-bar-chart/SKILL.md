# 麦肯锡条形图制作 Skill

## 概述

条形图(Bar Chart)是麦肯锡最常用的图表类型之一，特别适合展示类别对比和排名。本Skill提供完整的麦肯锡条形图设计规范。

## 条形图适用场景

### 最适合
- 类别数量较多(5-15个)
- 类别名称较长
- 需要展示排名
- 横向对比更直观

### 不适合
- 时间序列数据(用柱状图)
- 类别少于3个
- 需要展示趋势变化

## 麦肯锡条形图设计规范

### 1. 基本结构

```
┌────────────────────────────────────────────────────┐
│ 标题: [核心观点陈述]                                 │
├────────────────────────────────────────────────────┤
│                                                    │
│ 类别A ████████████████████████████████████ 89%    │
│ 类别B ████████████████████████████████ 76%        │
│ 类别C ██████████████████████████ 65%              │
│ 类别D ████████████████████ 54%                    │
│ 类别E ████████████████ 43%                        │
│                                                    │
├────────────────────────────────────────────────────┤
│ 图例: ■ 类别1  ■ 类别2  数据来源: XXX              │
└────────────────────────────────────────────────────┘
```

### 2. 排序规则

**规则1: 按数值排序（推荐）**
```
从大到小排列，便于快速比较

示例:
最高优先级 ████████████████████ 89%
高优先级   ████████████████ 76%
中优先级   ████████████ 65%
低优先级   ████████ 54%
最低优先级 ████ 43%
```

**规则2: 按逻辑顺序**
```
当类别有固定顺序时（如时间、流程）

示例:
第一阶段 ████████████████████
第二阶段 ████████████████
第三阶段 ████████████
第四阶段 ████████
```

**规则3: 字母顺序**
```
当需要查找特定类别时

示例:
产品A ████████████████████
产品B ████████████████
产品C ████████████
产品D ████████
```

### 3. 颜色配置

**麦肯锡蓝色系**:
```css
/* 主色 */
--mckinsey-primary: #051C2C;    /* 深蓝 */
--mckinsey-secondary: #1E3A5F;  /* 中蓝 */
--mckinsey-tertiary: #3A5F8A;   /* 浅蓝 */

/* 对比色 */
--mckinsey-black: #000000;      /* 黑色（对比） */
--mckinsey-gray: #666666;       /* 灰色（辅助） */
```

**双色对比（时间对比）**:
```
4月数据: ■■■■■■■■ 黑色 (#000000)
8月数据: ■■■■■■■■ 蓝色 (#051C2C)
```

**多色区分（类别区分）**:
```
类别A: ■■■■ 深蓝 (#051C2C)
类别B: ■■■■ 中蓝 (#1E3A5F)
类别C: ■■■■ 浅蓝 (#3A5F8A)
类别D: ■■■■ 灰色 (#666666)
```

### 4. 数据标签

**位置标准**:
```
方式1: 条形末端（推荐）
类别A ████████████████████████ 89%

方式2: 条形内部（长条形）
类别A ██████████89%████████████

方式3: 条形末端外侧（短条形）
类别A ████ 23%
```

**格式标准**:
```
- 整数不带小数: 89%
- 保留一位小数: 89.5%
- 保持一致性: 所有标签格式统一
```

### 5. 间距与比例

```
条形宽度: 30-40px
条形间距: 10-15px
边距: 20-30px

比例关系:
条形宽度 : 间距 = 2.5 : 1
```

## 对比型条形图

### 时间对比条形图

**结构**:
```
        4月    8月
类别A  ■■■■■  ████████  89%
类别B  ■■■■   ███████   76%
类别C  ■■■    █████     65%

图例: ■ 4月数据  █ 8月数据
```

**设计要点**:
```
1. 两个时间点使用对比色
2. 新数据放在右侧/上方
3. 保持对齐便于比较
4. 标注增长/下降
```

### 分组条形图

**结构**:
```
        2022    2023
类别A  ███ ████  增长20%
类别B  ██ ███   增长15%
类别C  ███ ███  持平

图例: █ 2022  █ 2023
```

## 代码示例

### Python (Matplotlib)

```python
import matplotlib.pyplot as plt
import numpy as np

# 数据
categories = ['类别A', '类别B', '类别C', '类别D', '类别E']
values = [89, 76, 65, 54, 43]

# 麦肯锡颜色
mckinsey_blue = '#051C2C'

# 创建图表
fig, ax = plt.subplots(figsize=(10, 6))

# 绘制条形图
bars = ax.barh(categories, values, color=mckinsey_blue, height=0.6)

# 添加数据标签
for bar, value in zip(bars, values):
    ax.text(bar.get_width() + 2, bar.get_y() + bar.get_height()/2,
            f'{value}%', va='center', fontsize=12)

# 样式设置
ax.set_xlim(0, 100)
ax.spines['top'].set_visible(False)
ax.spines['right'].set_visible(False)
ax.set_xlabel('')

plt.tight_layout()
plt.show()
```

### Python (Plotly)

```python
import plotly.graph_objects as go

# 数据
categories = ['类别A', '类别B', '类别C', '类别D', '类别E']
values = [89, 76, 65, 54, 43]

# 创建条形图
fig = go.Figure(go.Bar(
    x=values,
    y=categories,
    orientation='h',
    marker_color='#051C2C',
    text=[f'{v}%' for v in values],
    textposition='outside'
))

# 更新布局
fig.update_layout(
    title='核心观点标题',
    xaxis_range=[0, 100],
    showlegend=False,
    height=400,
    margin=dict(l=20, r=20, t=40, b=20)
)

fig.show()
```

## 何时不要使用

- 需求仅是简单文案润色或排版修饰，不涉及本 Skill 对应的方法或图表决策。
- 用户问题与该 Skill 的核心场景不匹配，存在更直接的专项 Skill 可用。
- 仅需一次性小修改且不需要此 Skill 的方法论框架。

## 快速使用清单

- 明确页面目标或分析问题（先结论后图表）。
- 确认输入数据字段、时间口径、分组维度。
- 选择图表类型或方法模板并执行。
- 产出标题、关键结论、图表与数据来源。
- 用本 Skill 的质量检查项完成最终自检。

## 常见错误

- 图表类型与业务问题不匹配，导致结论表达失焦。
- 标题不是结论句，只写成章节名。
- 过度堆叠信息，单页承载超过可读上限。
- 缺失数据口径和来源说明，影响复用与可信度。
