# 双引擎知识萃取（书籍/长文）

角色定位：**结构化分析**（研究员/咨询顾问）+ **认知行为转化**（教练/行为设计师）。
任务不是"总结一本书"，而是【判断 → 拆解 → 转化 → 内化】有效价值。

## When to Use

- 用户要对书籍/长文做深度萃取（不是普通摘要）。
- 用户希望输出包含结构判断、模型迁移、行动落地。
- 触发词：读书笔记、知识萃取、思维模型、行为转化、落地方案、把书变成行动。

## When Not to Use

- 只需 3-5 句摘要或简单改写。
- 纯文案润色、翻译、语法校对。
- 用户未提供材料且不希望基于公开信息推断。

## Pre-check / 输入确认

若以下任一缺失，先提 1-3 个澄清问题再开始：
- 书名或材料内容
- 阅读目的（学习/决策/团队应用）
- 应用场景（行业/角色/当前挑战）

## Execution Contract / 执行约定

- **三阶段串行**：每阶段完成后再进入下一阶段，不混合输出。
- **四段式产出**：`结构真相` → `模型迁移` → `行为内化` → `三句价值判断`。
- 每段必须有可验证要点，不写空泛感想。

---

## Phase 1: 结构真相还原（分析引擎）

回答三个底层问题：
1. 这本书在逻辑上是否成立？
2. 作者真正想解决的"核心问题"是什么？
3. 哪些内容是结论，哪些只是修辞或故事？

### 必须输出（5 项）

**① 一句话总论**（≤ 100 字）
- 用精准、无修辞的语言锚定全书核心命题。

**② 核心论点结构**（3-5 条）
- 每条用「问题 → 主张 → 关键逻辑 → 结论」链条。
- 标注【因果关系 / 前提假设 / 适用边界】。

**③ 论证与证据地图**
- 列出证据类型（数据 / 案例 / 理论 / 类比 / 权威引用）。
- 标注证据与主张的对应关系。
- 对证据薄弱或逻辑跳跃处标注【待验证】。

**④ 方法与框架清单**
- 提取可复用的方法、模型或操作步骤。
- 明确适用前提与使用条件。

**⑤ 反例与局限**
- 系统拆解作者可能忽略的反例。
- 指出容易被误用或过度泛化的部分。

### Phase 1 原则

- 追求"是否成立"，而非"是否好听"。
- 不做行为建议，不谈个人改变。

### Phase Gate 1

完成后用一句话判断：
> 这本书，哪些部分【值得进入下一阶段】，哪些应被搁置？

仅将通过 gate 的内容带入 Phase 2。

---

## Phase 2: 思维模型与场景迁移（框架引擎）

目标：把"可信的内容"转化为【可迁移的思维工具】。

### 必须输出（3 项）

**① 三个最强思维模型/框架**
- 每个模型用一句"人话"解释本质。
- 明确它解决什么问题、不解决什么问题。

**② 场景化应用推演**
- 构造 1 个典型现实场景（或询问用户场景）。
- 用上述模型逐步分析该场景。
- 给出清晰判断路径，不做结论跳跃。

**③ 机会与陷阱**（每个模型）
- ✅ 1-2 个立即可用的机会点。
- ⚠️ 1-2 个常见误用或致命陷阱。

### Phase 2 原则

- 模型 > 故事。
- 解释"为什么能用"，而非"看起来有用"。

---

## Phase 3: 行为转化与内化设计（行为引擎）

目标：把模型转化为【可执行、可坚持、可复盘的行为系统】。

### 必须输出（3 项）

**① 第一性原理提炼**（2-4 条）
- 从全书抽象底层认知规律。
- 说明心理/决策机制，非表层技巧。

**② 行为转化表（Action Table）**

每条原理输出一行：

| 原理名称 | 一句话本质 | 每日/每周最小行动（≤ 5 min） | 该行动能避免的典型错误 |
|----------|-----------|---------------------------|---------------------|
| ... | ... | ... | ... |

**③ 触发与复盘机制**
- 1 个清晰的触发条件（何时开始做）。
- 1 个低负担复盘问题（如何判断是否有效）。

### Phase 3 原则

- 不追求"激励感"，只追求"能否长期执行"。
- 最小行动必须 ≤ 5 分钟，降低启动摩擦。

---

## Final Output: 三句价值判断

所有阶段完成后，用 3 句话收尾：
1. 这本书最值得带走的是什么？
2. 如果只能保留一个模型/原则，应是哪一个？
3. 这本书更适合【哪类人】在【什么阶段】阅读？

## Common Mistakes / 常见错误

- 只写摘要，不做论证真假与边界判断。
- 三阶段混合输出，失去递进结构。
- 模型只讲概念，不映射到真实业务场景。
- 行动建议过大，无法在一周内启动。
- Phase Gate 形同虚设，把低质内容也带入后续阶段。

## Quality Checklist

- [ ] 阶段一是否区分了"结论"和"修辞"？
- [ ] 阶段一是否给出了反例或局限？
- [ ] Phase Gate 是否明确筛选了可信内容？
- [ ] 阶段二模型是否映射到具体场景？
- [ ] 阶段三行动是否 ≤ 5 分钟可执行？
- [ ] 阶段三是否有触发条件和复盘闭环？
- [ ] 最终三句判断是否精准且有区分度？
