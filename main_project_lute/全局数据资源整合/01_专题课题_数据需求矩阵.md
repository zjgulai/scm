### 数仓开发说明（与上表对应）

- **粒度、主键、分区**：每张表的**粒度（一行代表什么）、主键（含联合主键）、分区键**已在 [05_数仓表结构与主键设计.md](05_数仓表结构与主键设计.md) 中逐表定义，数仓建表时以 05 为准，避免重复行或 Join 歧义。
- **fact_***：事实表，分区键一般为日期/月份；主键见 05（如 fact_order 主键 order_id，fact_return 主键 (return_id, return_line_id) 或 (return_id, sku_id)，汇总表为多列联合主键）。
- **dim_***：维度表，以业务主键为 PK（如 channel_id, warehouse_id, return_reason_code）；若需历史拉链可增加 surrogate_key + effective_dt/end_dt。
- **ods_***：贴源层，主键与分区见 05；外部数据先落 ODS 再加工到 fact/dim。
- **字段英文名**：上表仅列出关键字段，建表时可按业务再增补；命名风格建议统一（国家→country_code，渠道→channel_id，金额→*_amt，数量→*_qty，率→*_rate/*_pct）。
- **跨表关联**：**order_id** 贯穿 fact_order / fact_return / fact_order_item；**campaign_id** 贯穿 fact_campaign_* / fact_order（若有）；**country_code + channel_id** 贯穿渠道与营销层；VOC 与订单通过 **order_id / return_id / voc_ticket_id** 关联。完整逻辑外键见 05 第五节。

> 评审通过后可将本矩阵 + 05 作为数仓需求基线，按「库表建议」与 05 中的主键/粒度排期开发，分析侧即可无阻碍使用。
