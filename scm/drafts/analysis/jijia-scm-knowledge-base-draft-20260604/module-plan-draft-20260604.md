# 积加计划模块业务逻辑与指标血缘

## 模块定位

计划模块回答三个问题：

1. 什么时候会缺货。
2. 需要采购、调拨、发货还是装配。
3. 每个动作的数量和日期是什么。

它不是单纯的计划单据模块，而是连接销售预测、库存、采购交期、物流时效、仓库处理能力和安全库存策略的决策层。

## 核心子模块

| 子模块 | 业务作用 | 关键对象 |
|---|---|---|
| 智能补货 | 根据销量、库存和供应链周期推算补货数量 | 目标库存量、推荐采购量、推荐调拨量、缺货日期 |
| 红绿线 | 预测库存消耗曲线和缺货区间 | 缺货日期、缺货天数、补货数量、未入库供应链单据 |
| 连续补货建议看板 | 对销量平稳商品给出连续发货建议 | 建议发货量、建议发 FBA 仓日期、触发发货条件 |
| 采购计划 | 把补货需求转成采购计划单 | 计划单号、目的仓、物流方式、计划量、状态 |
| 发货计划 | 把补货需求转成发货排期 | 发货仓、目的仓、物流方式、出运日期、发货量 |
| 装配计划 | 管理组合品本地组装预装需求 | 计划需求量、实际需求量、可装量、装配中、已装配 |

## 业务逻辑

### 智能补货

智能补货的业务目标是让库存水位在不断货和不积压之间保持平衡。其逻辑是先计算目标库存量，再扣减已有库存、在途、已下单和可调拨资源，最后得出推荐采购量或推荐调拨量。

核心链路：

```text
历史销量/目标销量
  -> 日均消耗量
  -> 目标库存天数
  -> 目标库存量
  -> 扣减现有和在途库存
  -> 推荐采购量/推荐调拨量
```

关键参数：

- 缺货临界点：按断货判断或按安全库存判断。
- FBA 在途库存来源：ERP 单据在途或 FBA 报告在途。
- 是否纳入国内仓、海外仓、供应商仓的已下单、在途、可调拨。
- 共享库存处理规则。
- 历史销量是否剔除断货数据。
- 平台仓 Reserved FC processing 是否纳入补货良品可用量。

### 红绿线

红绿线用于解释缺货点和补货点。它把计划、采购订单、交货单、发货单、调拨单等未完成供应链单据放在同一个时间轴中，判断库存何时被消耗完，以及后续单据何时能补上。

关键逻辑：

- 未完成供应链单据包括采购计划、采购订单、交货单、发货单、调拨单。
- 预计入库日期按采购、质检、运输和入库处理环节累加。
- 补货数量按缺货起点缺口和缺货期间日均消耗推算。
- 物流方式推荐受缺货日期和各段时效影响。

### 连续补货建议

连续补货建议适用于销量相对平稳的商品。它按照连续补货规则，在未来一段日期范围内生成多个建议发货点，用来控制库存水位。

关键参数：

- 生成补货日期范围。
- 目标发货天数。
- 触发发货条件。
- 最小发货量。
- 单箱数量。
- 普通模式或海外中转模式。

### 采购计划

采购计划是采购环节的起点。它承接智能补货或人工计划结果，再通过分单转成采购订单、预装需求或委外订单。

核心状态：

- 草稿。
- 审核中。
- 审核驳回。
- 待下单或待分单。
- 下单中或分单中。
- 已完成。
- 已作废。

关键控制：

- 数据权限可按目的仓或目的仓+SKU过滤。
- 字段权限会影响采购成本、头程成本、总成本等金额字段可见性。
- 目的仓和物流方式会影响后续分单和物流计划。

### 发货计划

发货计划用于把补货需求转成物流排期。它综合发货仓、目的仓、物流方式、出运日期、发货量、预计入库日期，并沉淀重量、体积、装箱体积、装箱重量等物流准备字段。

核心状态：

- 草稿。
- 审核中。
- 审核驳回。
- 待处理。
- 已处理。
- 已作废。

### 装配计划

装配计划用于组合品在国内自营仓组装时的预装需求管理。它的核心不是采购子件，而是跟踪母件需求是否能被子件库存满足。

关键指标：

- 可装量：装配仓内可用于组装组合品的数量。
- 计划需求量：来自采购计划。
- 实际需求量：可手动调整。
- 装配中：已生成装配单但未完成。
- 已装配：装配单已完成数量。
- 预计装配时间：分单日期、最长子件交期、质检时间和入库处理时间共同决定。

## 指标字典

| 指标编码 | 指标 | 粒度 | 口径 |
|---|---|---|---|
| SCM-PLAN-001 | 日均消耗量 | MSKU + 站点 + 日期 | 历史订单或目标销量规则计算，可按 3/7/14/30/60/90 天权重 |
| SCM-PLAN-002 | 目标库存天数 | MSKU + 目的仓 | 计划审批、采购交期、质检、物流、入库处理、安全库存、补货周期等周期之和 |
| SCM-PLAN-003 | 目标库存量 | MSKU + 目的仓 | 目标库存天数 × 日均消耗量 |
| SCM-PLAN-004 | 推荐采购量 | SKU/MSKU + 目的仓 | 目标库存量扣减良品可用、可调拨、已下单、计划入库、待出运、在途等 |
| SCM-PLAN-005 | 推荐调拨量 | SKU/MSKU + 调出仓 + 目的仓 | 目标库存量扣减目的仓库存和可覆盖在途后的调拨需求 |
| SCM-PLAN-006 | 缺货日期 | MSKU + 目的仓 | 库存消耗到缺货临界点的日期 |

| SCM-PLAN-008 | 缺货量 | MSKU + 目的仓 | 缺货期间目标销量扣减可用库存和计划入库后的缺口 |
| SCM-PLAN-009 | 建议发货量 | MSKU + 目的仓 + 发货点 | 连续补货规则下生成的发货数量 |
| SCM-PLAN-010 | 采购计划量 | 采购计划 + SKU | 计划单行的计划数量 |
| SCM-PLAN-011 | 发货计划量 | 发货计划 + SKU | 发货计划单据中的产品发货数量 |
| SCM-PLAN-012 | 装配计划未装量 | 装配计划 + SKU | 实际需求量 - 装配中 - 已装配 |

## 指标血缘

```text
销售订单/目标销量
  -> 日均消耗量
  -> 目标库存天数
  -> 目标库存量
  -> 推荐采购量/推荐调拨量
  -> 采购计划/发货计划/装配计划
  -> 分单/发货/装配后续链路
```

上游数据：

- 销售订单。
- 平台仓库存报告。
- 产品库存。
- 供应商产品采购交期。
- 物流方式时效。
- 系统参数。

下游数据：

- 采购计划。
- 发货计划。
- 装配计划。
- 调拨单。
- 采购订单。
- 发货单。

## 数据建模建议

| 表 | 粒度 | 必备字段 |
|---|---|---|
| `fact_replenishment_snapshot` | 计算批次 + MSKU + 目的仓 | 目标库存天数、日均消耗量、目标库存量、推荐采购量、推荐调拨量、缺货日期 |
| `fact_purchase_plan_line` | 采购计划 + SKU/MSKU | 计划量、分单量、目的仓、物流方式、状态、创建人 |
| `fact_shipping_plan_line` | 发货计划 + SKU/MSKU | 发货仓、目的仓、物流方式、出运日期、发货量、预计入库日期 |
| `fact_assembly_plan_line` | 装配计划 + 组合品 | 可装量、计划需求量、实际需求量、装配中、已装配 |
| `meta_parameter_snapshot` | 参数批次 | 缺货临界点、FBA在途来源、共享库存规则、销量规则 |

## 后续核验清单

- 公司是否使用智能补货作为采购计划来源。
- FBA 在途口径实际选择 ERP 单据还是平台报告。
- 历史销量是否剔除断货数据。
- 目标库存天数中各项默认值是否被公司修改。
- 发货计划是否被物流团队实际使用，还是直接使用发货单。
- 装配计划是否覆盖所有组合品，是否存在绕过预装需求的线下流程。
