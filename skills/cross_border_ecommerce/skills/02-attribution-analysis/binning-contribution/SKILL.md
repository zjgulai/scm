# 跨境电商分箱贡献分析 Skill

## 概述

分箱贡献分析是精细化运营的重要工具。本Skill提供系统化的分箱分析方法，帮助用户：

- **灵活分箱**: 按任意维度（客单价、客品数、时间等）划分区间
- **量化贡献**: 计算各区间对整体指标的贡献度
- **识别关键区间**: 发现高贡献区间和问题区间
- **支持决策优化**: 基于分箱结果制定针对性策略

### 核心公式

**静态贡献计算**:
```
单个区间对整体指标的贡献:
Contribution_i = w_i × M_i

其中:
- w_i = 区间i的权重（销售额占比/订单数占比等）
- M_i = 区间i的指标值
```

**动态变动贡献**:
```
单个区间对整体变动的贡献:
Contribution_Δi = Δw_i × M_i上期 + w_i本期 × ΔM_i

= 结构变动贡献 + 自身变动贡献
```

## 核心功能

### 1. 分箱策略

#### 1.1 等频分箱（推荐）

**原理**: 每个区间的样本量相近

**适用场景**:
- 数据分布偏态（少数极高值，大部分偏低）
- 需要每个区间有足够统计意义
- 跨境电商订单数据（推荐）

**实现方法**:
```python
# 等频分箱（分位数分箱）
bins = pd.qcut(data, q=5, labels=['极低', '低', '一般', '中高', '高'])
```

**优点**: 每个区间样本量均衡，统计意义强

**缺点**: 区间宽度可能不一致

#### 1.2 等距分箱

**原理**: 每个区间的宽度相等

**适用场景**:
- 数据分布均匀
- 需要区间边界有明确业务含义
- 有明确的业务分界点

**实现方法**:
```python
# 等距分箱
bins = pd.cut(data, bins=5, labels=['极低', '低', '一般', '中高', '高'])
```

**优点**: 区间宽度一致，易于理解

**缺点**: 偏态分布时某些区间样本量过少

#### 1.3 自定义分箱

**原理**: 根据业务经验自定义区间边界

**适用场景**:
- 有明确的业务分界标准
- 双峰分布或多峰分布
- 需要符合业务习惯

**示例**:
```python
# 自定义分箱
bins = pd.cut(data, bins=[0, 20, 50, 90, 150, float('inf')],
              labels=['极低', '低', '一般', '中高', '高'])
```

### 3. 贡献度计算

#### 3.1 静态贡献

**计算步骤**:
```
1. 计算各区间基础指标: M_i = P_i / S_i
2. 计算各区间权重: w_i = S_i / S_总
3. 计算各区间贡献: Contribution_i = w_i × M_i
4. 计算贡献占比: Contribution_ratio_i = Contribution_i / M_总 × 100%
```

**验证**: 所有区间贡献占比之和 = 100%

#### 3.2 动态变动贡献

**结构变动贡献**:
```
Structure_Contribution_i = Δw_i × M_i上期
```

**自身变动贡献**:
```
Self_Contribution_i = w_i本期 × ΔM_i
```

**总贡献**:
```
Total_Contribution_i = Structure_Contribution_i + Self_Contribution_i
```

**判断逻辑**:
```
结构变动贡献占比大 → 调整运营策略
自身变动贡献占比大 → 优化成本定价
```

### 4. 高贡献区间识别

**识别标准**:
```
高贡献区间: Contribution_ratio_i > 平均贡献占比
问题区间: Contribution_ratio_i < 平均贡献占比 且 M_i < M_总
机会区间: Contribution_ratio_i 低 但 增长潜力大
```

## 使用方法

### 输入数据要求

**订单级数据格式**:
```csv
order_id,customer_id,order_amount,item_count,gross_profit,order_date
001,C001,180,5,72,2024-01-01
002,C002,45,1,5,2024-01-01
003,C003,220,8,99,2024-01-01
...
```

**聚合数据格式**:
```csv
bin_name,period,sales,gross_profit,orders
高客单,2024-01,2400000,912000,8000
中高客单,2024-01,2600000,819000,13000
...
```

### 分析流程

```
步骤1: 数据加载
- 读取订单级或聚合数据
- 确定分箱维度

步骤2: 分箱处理
- 选择分箱策略（等频/等距/自定义）
- 执行分箱
- 检查各区间样本量

步骤3: 计算基础指标
- 计算各区间销售额/毛利/订单数
- 计算各区间指标值（毛利率等）

步骤4: 计算贡献度
- 计算静态贡献
- 如需对比，计算动态变动贡献

步骤5: 识别关键区间
- 识别高贡献区间
- 识别问题区间
- 识别机会区间

步骤6: 输出报告
- 生成贡献度分析表
- 输出优化建议
```

## 示例场景

### 示例1: 客单价分箱优化

**场景**: 电商毛利率需要提升，分析客单价区间贡献

**当前状态**:
- 整体毛利率: 22.24%
- 高客单占比: 24%
- 高客单贡献: 45%

**优化方案**:
```
目标: 将高客单占比从24%提升至30%

预期效果:
- 新增高客单销售: 60万
- 新增高客单毛利: 22.8万
- 整体毛利率提升: 22.24% → 24.5%

运营动作:
1. 推出高客单组合套装
2. 满减活动引导凑单
3. 高客单专属优惠券
```

### 示例2: 客品数分箱诊断

**场景**: 便利店毛利率下滑，诊断客品数区间

**问题定位**:
```
毛利率从21.5%下滑到20.7%，下滑0.8pp

根因: 极低客品数（1件/单）占比从12%升至18%
- 极低客品数毛利率: 14%
- 拖累整体毛利: -1.04pp
```

**改进措施**:
```
1. 满减活动: 满2件减5元
2. 组合推荐: 购买此商品的顾客还买了...
3. 会员专享: 多件购买享折扣
4. 临期提醒: 顺路零买转囤货
```

## 注意事项

### 分箱策略选择

| 数据特征 | 推荐策略 | 原因 |
|---------|---------|------|
| 高度偏态 | 等频分箱 | 保证各区间样本量 |
| 均匀分布 | 等距分箱 | 区间宽度一致 |
| 多峰分布 | 自定义分箱 | 按峰值划分 |
| 有业务标准 | 自定义分箱 | 符合业务习惯 |

### 常见问题

**Q: 分箱数量如何确定？**
A: 一般5-10个区间，保证每个区间有足够样本量（建议>5%占比）

**Q: 区间边界如何定义？**
A: 等频分箱自动计算，自定义分箱结合业务经验

**Q: 动态分析一定要拆分结构和自身变动吗？**
A: 是的，这是精准定位问题的关键

### 最佳实践

1. **先看分布再选策略**: 绘制直方图了解数据分布
2. **验证区间合理性**: 每个区间样本量要足够
3. **结合业务解读**: 数字背后要有业务逻辑
4. **转化为行动**: 分箱结果要指向具体改进措施
