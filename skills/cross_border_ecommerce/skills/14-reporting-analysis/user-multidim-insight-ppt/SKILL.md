# 用户多维度洞察与管理层PPT

## 快速概览

| 项目 | 内容 |
| --- | --- |
| **输入** | 结构化 Excel（含时间窗口、区域、新老客、销售额/销量/费用等字段） |
| **输出** | 分析脚本 + 多 Sheet 结果 Excel + Markdown 洞察报告 + 管理层汇报 PPT |
| **核心能力** | 多维指标计算、资源匹配度评估、价值产出效率分析、结论先行表达与 PPT 生成框架 |
| **典型场景** | 用户多维度洞察、区域 × 新老客交叉分析、预算再分配评估、管理层季度/阶段性复盘 |
| **依赖示例** | pandas、openpyxl、matplotlib、python-pptx（可按项目实际调整） |

## 适用范围

用于把一份 Excel 分析任务沉淀成四类交付物：

1. 分析脚本
2. 结果 Excel
3. Markdown 洞察报告
4. 管理层 PPT

默认适合以下问题：
- 基于时间窗口对比多维指标
- 评估区域 / 新老客 / 渠道等交叉维度表现
- 评估资源匹配度与价值产出效率
- 把分析结果升级为管理层可读的图表汇报

## 推荐放置位置

若项目已有 `data_example/` 风格目录，默认沿用：

- 输入：`data_example/data/*.xlsx`
- 脚本：`data_example/scripts/*.py`
- 输出：`data_example/outputs/*`

## 标准工作流

### Step 1：先做数据勘探和口径确认

先确认 4 件事：

1. 时间窗口是否真实存在于目标 Sheet
2. 维度是否齐全，例如区域、新老客、整体口径
3. 用户给的公式是否与源数据字段正负号一致
4. 是否需要补算汇总层，例如“整体-新客”“整体-老客”

若用户口径与数据实际不一致，必须先确认，再实现。

## Step 2：构建分析脚本

分析脚本至少包含这些模块：

1. 读取指定 Sheet
2. 数值字段类型清洗
3. 补算缺失的汇总维度
4. 计算衍生指标
5. 生成基线 vs 对比窗口矩阵
6. 评估资源匹配度
7. 输出 Excel / Markdown / 校验文件

推荐拆分函数：

```python
load_data()
append_total_segment_rows()
compute_metrics()
build_detail_sheet()
build_comparison_sheet()
build_resource_match_sheet()
build_promo_mix_sheet()
build_validation_lines()
write_outputs()
```

## Step 3：指标设计

若任务是“用户多维度洞察”，优先使用以下结构：

### 核心输入指标

- 购买人数
- 订单数
- 销售额
- 销量
- 广告费
- gtm推广费
- 折扣额
- 用户ARPU值

### 常用衍生指标

```text
总推广费 = 广告费 + |gtm推广费| + |折扣额|
ARPU值 = 用户ARPU值
人均推广费 = 总推广费 / 购买人数
人均毛利额 = (销售额 - 总推广费) / 购买人数
人均订单数 = 订单数 / 购买人数
人均销售额 = 销售额 / 购买人数
人均毛利率 = 人均毛利额 / 人均销售额
人均订单价 = 人均销售额 / 人均订单数
人均品单价 = 销售额 / 销量
人均客品数 = 销量 / 订单数
```

如果源表中的费用字段是负数记账口径，优先与用户确认是否取绝对值后再汇总。

## Step 4：资源匹配度分析

最小可用框架：

1. 选取原子交叉维度，例如 `北美-新客`、`北美-老客`、`欧洲-新客`、`欧洲-老客`
2. 计算投入占比：`总推广费 / 原子维度总推广费合计`
3. 计算产出占比：`经营毛利额 / 原子维度经营毛利额合计`
4. 计算资源匹配差：`产出占比 - 投入占比`
5. 计算每元推广费产出毛利额：`经营毛利额 / 总推广费`

四象限命名统一使用：

- 高投入高产出
- 高投入低产出
- 低投入高产出
- 低投入低产出

## Step 5：洞察报告结构

默认用五幕故事线：

1. 整体健康度速览
2. 区域差异
3. 新老客结构
4. 交叉维度下钻
5. 行动建议

要求：

- 标题和每段首句优先写结论
- 每个结论后面跟 1 个明确数据支撑
- 管理建议只保留 3 到 4 条，避免发散

## Step 6：管理层PPT结构

默认输出 5 页：

1. 执行摘要页
2. 整体效率页
3. 区域与新老客对比页
4. 资源匹配四象限页
5. 费用结构与行动建议页

标题必须是完整结论句，不要写成“分析结果”“数据概览”。

### 母婴品牌出海示例（可直接套用）

当分析对象是母婴品牌出海（如北美 / 欧洲独立站）时，建议：

- 时间窗口：用 YTD / MAT 口径对比不同阶段投放与利润表现
- 区域维度：至少区分北美 / 欧洲 / 整体
- 客群维度：至少区分新客 / 老客 / 整体

示例标题与要点：

1. 执行摘要页  
   - 标题示例：**“欧洲独立站以更低获客成本撬动更高人均毛利，老客是利润核心池”**  
   - 要点：  
     - 欧洲站人均推广费 vs 北美站对比（低多少）  
     - 欧洲站人均毛利额 vs 北美站对比（高多少）  
     - 老客 ARPU / 订单数倍数 vs 新客

2. 区域与新老客页  
   - 标题示例：**“欧洲线下折扣依赖更高，但整体利润率仍高于北美”**  
   - 要点：  
     - 各区域人均推广费 / 人均毛利额 / 人均毛利率  
     - 新客 vs 老客在人均推广费、人均毛利额上的差异

3. 资源匹配度页  
   - 标题示例：**“欧洲新客是最优加码对象，北美新客是当前预算优化主战场”**  
   - 要点：  
     - `欧洲-新客` 投入占比 vs 产出占比、资源匹配差  
     - `北美-新客` 资源匹配差为负，说明投入偏高、产出偏低

## 与其他Skills协同

在完整项目中，推荐与以下 Skills 配合使用：

- **cbec-yoy-mom-analysis**（同比环比分析）  
  - 在 Step 2/3 之前，用于确认时间窗口定义与同比/环比口径，避免窗口含义混淆。

- **cbec-margin-attribution**（毛利率归因）  
  - 当管理层追问“为什么人均毛利率变化”时，用该 Skill 对毛利率波动做进一步拆解（成本率 / 广告费率 / 促销折扣等）。

- **cbec-data-reporting**（数据报告撰写）  
  - 当输出形式为长版分析报告（而非 PPT）时，用其提供的报告结构与表达模板，承接本 Skill 的结果数据。

- **cbec-dashboard-design**（数据看板设计）  
  - 当用户希望把本 Skill 的关键指标长期可视化监控时，用其设计包含 ARPU、人均推广费、人均毛利额、资源匹配度的标准看板。
