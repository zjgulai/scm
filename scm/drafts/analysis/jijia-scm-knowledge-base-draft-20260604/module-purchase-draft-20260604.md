# 积加采购模块业务逻辑与指标血缘

## 模块定位

采购模块负责把计划需求转成供应商履约行为，并沉淀采购交付、质量、入库和供应商对账数据。

它的主线是：

```text
采购计划
  -> 分单任务
  -> 采购订单/预装需求/委外订单
  -> 交货单
  -> 发货单/头程单/入库单
  -> 供应商对账/供应商仓货权
```

## 核心子模块

| 子模块 | 业务作用 | 关键对象 |
|---|---|---|
| 新版分单 | 将采购计划分配给供应商或装配仓 | 需求量、分单数量、供应商、装配仓 |
| 采购订单 | 记录供应商采购执行 | 采购量、交货量、未交量、入库量、金额 |
| 交货单 | 记录供应商交货和质检发货 | 实交量、质检状态、待发货、已发货、入库中 |
| 供应商产品 | 维护报价、采购交期和默认供应商 | 采购交期、MOQ、含税/不含税价格 |
| 供应商对账 | 对采购、发货、入库节点进行货款核对 | 采购量、发货量、入库量、金额 |
| 供应商仓货权 | 管理供应商仓库存货权转移和请款节点 | 调拨出运、装配入库实物类型、请款记录 |

## 业务逻辑

### 分单

分单是采购计划的下一步。非组合品通常生成普通采购订单。组合品存在两条路径：

- 按母件采购：直接采购组合品母件。
- 按子件采购：子件生成采购订单，母件根据装配仓生成预装需求或委外订单。

关键逻辑：

- 非组合品需求量 = 计划量 - 已分单量。
- 组合品装配需求量 = 计划量 - 已分单量。
- 已分单量在组合品场景中可能由委外采购量和预装需求预装量共同构成。
- 如果 SKU 只有一个供应商报价，系统可自动分配采购任务。
- 如果存在多个供应商报价，默认供应商优先；没有默认供应商时，需要核验实际系统取值规则。

### 采购订单

采购订单是供应商履约的核心事实表。它记录采购数量、交货数量、入库数量、退货换货数量和货值。

关键数量：

- 总采购量。
- 交货量。
- 实交量。
- 未交量。
- 入库量。
- 入库良品量。
- 入库次品量。
- 未入库量。
- 入库前退货量。
- 入库后退货量。
- 入库后换货量。

关键金额：

- 含税货值。
- 不含税货值。
- 可退税金额。
- 退税后货值。
- 采购其他费用。
- 订单总金额。
- 已交含税货值。
- 入库含税货值。

### 交货单

交货单表示供应商实际交货节点。它连接采购订单、质检、发货和入库。

关键状态：

- 待质检。
- 质检中。
- 待发货。
- 已发货。
- 入库中。
- 已完成。
- 已作废。

重要规则：

- 工厂质检和仓库质检影响交货单状态。
- 国内自营仓可在交货单确认发货后自动生成发货单。
- 海外仓需要在发货单模块创建发货单并关联交货单。
- FBA 仓通常通过头程单和货件关联。
- 多个交货单生成同一发货单时，需要满足同供应商、同交货仓等条件。

### 供应商产品

供应商产品是采购执行的基础配置。指标体系中必须把它作为维表或主数据来源，而不是临时字段。

关键字段：

- 供应商。
- SKU。
- 采购员。
- 是否默认供应商。
- 采购交期。
- 最小起订量。
- 供应商货号。
- 状态。
- 币种。
- 实际税率。
- 发票类型。
- 票面税率。
- 退税率。
- 不含税单价。
- 含税单价。

### 供应商对账

供应商对账支持以采购订单、发货单、入库单维度核对货款。它的指标设计不能只看采购订单金额，还要结合货权转移节点和结算方式。

关键口径：

- 采购量：采购订单维度的采购数量。
- 发货量：实际发出数量。
- 入库量：实际完成接收数量。
- 入库金额：按入库量和单价计算。
- 对账单据号：可能是采购订单、发货单或入库单。

### 供应商仓货权

供应商仓货权适用于供应商仓采购入库不立即请款，延迟到调拨发货出运等节点才转移货权的业务。它受参数控制，并且要求调拨出库实物类型满足特定条件。

建模上必须单独保留：

- 货权是否已转移。
- 货权转移来源单据。
- 调出仓是否供应商仓。
- 出库实物类型。
- 请款节点。

## 指标血缘

```text
供应商产品
  -> 采购交期、默认供应商、价格
采购计划
  -> 分单任务
  -> 采购订单
  -> 交货单
  -> 发货单/入库单
  -> 供应商对账
```

上游数据：

- 采购计划。
- 供应商资料。
- 供应商产品。
- 产品资料。
- 装配仓配置。

下游数据：

- 采购订单。
- 交货单。
- 发货单。
- 入库单。
- 质检单。
- 供应商对账。
- 供应商仓货权。

## 数据建模建议

| 表 | 粒度 | 必备字段 |
|---|---|---|
| `dim_supplier` | 供应商 | 供应商编码、结算方式、采购员、状态 |
| `dim_supplier_product` | 供应商 + SKU | 采购交期、MOQ、默认供应商、报价、税率 |
| `fact_split_task_line` | 分单任务 + SKU + 供应商/装配仓 | 需求量、分单数量、生成单据类型 |
| `fact_purchase_order_line` | PO + SKU | 采购量、交货量、入库量、良品、次品、金额 |
| `fact_delivery_order_line` | LN + SKU | 实交量、质检状态、发货状态、入库状态 |
| `fact_supplier_reconciliation_line` | 供应商 + 单据 + SKU | 采购量、发货量、入库量、金额、对账状态 |
| `fact_supplier_ownership_transfer` | 供应商仓 + 调拨单 + SKU | 货权转移数量、转移节点、请款状态 |

## 后续核验清单

- 采购计划是否全部经过分单，是否存在手工采购订单。
- 分单列表导出字段是否包含生成单据类型。
- 公司是否使用默认供应商自动分配。
- 采购订单和交货单是否支持超交、少交、换货。
- 供应商对账采用采购订单、发货单还是入库单维度。
- 供应商仓货权参数是否启用。
