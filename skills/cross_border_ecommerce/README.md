## 项目概述

### 项目背景

母婴跨境电商企业面临着复杂的数据分析挑战：
- **生命周期驱动**: 宝宝成长阶段决定需求品类，客户需求随时间自然变化
- **安全性优先**: 妈妈群体高度关注产品安全、成分、认证
- **社群影响大**: 妈妈群、KOL种草对购买决策影响显著
- **多维度数据**: 销售、库存、成本、广告、物流等多源数据
- **多市场运营**: 不同国家/地区的货币、税率、法规差异
- **快速变化**: 市场需求、竞争态势、平台规则持续演变
- **精细化运营**: 需要深度的指标归因、成本控制和盈利分析

### 项目目标

构建一套专业的母婴跨境电商Claude Skills库，覆盖：
1. **数据分析基础能力**: 数据清洗→特征工程→模型创建→结果验证
2. **母婴专属分析**: 宝宝生命周期、品类迁移、订阅制、保质期管理
3. **跨境电商专门分析**: 财务、归因、成本、产品、趋势、预测、监控等12大领域
4. **外部知识增强**: 通过Biji知识库API获取行业最佳实践
5. **可复用的工作流**: 针对典型业务场景的完整解决方案
6. **质量保障体系**: 统一的输出标准和质量评分机制

### 核心价值

- **母婴专业化**: 深度融入母婴行业特性，提供专业洞察
- **标准化分析流程**: 确保分析质量和一致性
- **提升分析效率**: 自动化重复性工作，聚焦洞察
- **降低技术门槛**: 让业务人员也能进行专业分析
- **沉淀最佳实践**: 将专家经验转化为可复用资产
- **知识库增强**: 外部知识支持，持续优化分析方法
- **质量保障**: 统一输出标准，确保专业水准

## 数据分析基础Skills

### 1. 数据清洗 (data-cleaning)

**Skill名称**: `cbec-data-cleaning`  
**触发条件**: 用户上传数据文件并提到"清洗"、"清理"、"预处理"、"数据质量"等

**核心功能**:
- 缺失值检测与处理（删除/填充/插值）
- 异常值识别与处理（IQR/Z-score/孤立森林）
- 重复数据检测与去重
- 数据类型转换与标准化
- 格式统一（日期、货币、文本）

**处理流程**:
```
数据上传 → 质量评估 → 问题识别 → 清洗策略 → 执行清洗 → 质量验证 → 输出报告
```

**输出内容**:
- 清洗前后数据质量对比报告
- 清洗操作日志（记录所有修改）
- 清洗后数据集
- 数据质量评分

**示例触发语**:
- "帮我清洗这个销售数据文件"
- "数据里有缺失值，需要预处理"
- "检查数据质量并清理异常值"

- 统计显著性检验

**验证方法**:
```

```

**输出内容**:
- 验证报告（含统计检验结果）
- 模型置信区间
- 风险提示与使用建议

## 跨境电商专门Skills

### 财务分析类

#### 1. 杜邦分析法 (dupont-analysis)

**Skill名称**: `cbec-dupont-analysis`  
**触发条件**: 用户提到"杜邦"、"ROE分解"、"净资产收益率"、"财务拆解"等

**核心算法**:
```
ROE = 净利润率 × 总资产周转率 × 权益乘数

其中:
- 净利润率 = 净利润 / 营业收入
- 总资产周转率 = 营业收入 / 总资产
- 权益乘数 = 总资产 / 股东权益

扩展五因素模型:
ROE = (净利润/税前利润) × (税前利润/EBIT) × (EBIT/营业收入) × (营业收入/总资产) × (总资产/股东权益)
      = 税收负担 × 利息负担 × 营业利润率 × 资产周转率 × 权益乘数
```

**分析维度**:
| 维度 | 指标 | 业务含义 |
|------|------|---------|
| 盈利能力 | 净利润率 | 每元收入转化为利润的能力 |
| 运营效率 | 资产周转率 | 资产利用效率 |
| 财务杠杆 | 权益乘数 | 资本结构风险 |

### 指标归因分析类

#### 1. 贡献度计算 (contribution-calculation)

**Skill名称**: `cbec-contribution-calculation`  
**触发条件**: 用户提到"贡献度"、"贡献率"、"指标拆解"、"因素影响"等

**核心算法**:

**方法一: 因素分解法**
```
销售额 = 访客数 × 转化率 × 客单价

各因素贡献 = (本期值 - 上期值) × 其他因素基期值
```

**方法二: 链式替代法**
```
基期: V₀ = A₀ × B₀ × C₀
本期: V₁ = A₁ × B₁ × C₁

A因素贡献 = (A₁ - A₀) × B₀ × C₀
B因素贡献 = A₁ × (B₁ - B₀) × C₀
C因素贡献 = A₁ × B₁ × (C₁ - C₀)

验证: 总变动 = A贡献 + B贡献 + C贡献
```

**方法三: Shapley值法（博弈论）**
```
用于多因素交互作用的公平分配
φᵢ = Σ [|S|!(n-|S|-1)!/n!] × [v(S∪{i}) - v(S)]

其中:
- S: 不包含i的所有子集
- v(S): 子集S的协同价值
- n: 总因素数
```

**输出内容**:
- 各因素贡献度排序
- 贡献度可视化（瀑布图/帕累托图）
- 敏感性分析

#### 2. 单位经济模型 (unit-economics)

#### 2. 关联分析 (association-analysis)

**Skill名称**: `cbec-association-analysis`  
**触发条件**: 用户提到"关联分析"、"购物篮分析"、"捆绑销售"、"Apriori"等

**核心算法 - Apriori**:
```
支持度(Support) = P(A∪B) = 包含{A,B}的订单数 / 总订单数
置信度(Confidence) = P(B|A) = 包含{A,B}的订单数 / 包含A的订单数
提升度(Lift) = P(B|A) / P(B) = 置信度 / 支持度(B)

规则筛选:
- 最小支持度: 排除罕见组合
- 最小置信度: 确保规则可靠性
- 提升度>1: 表示正相关，有推荐价值
```

**业务应用**:
| 应用场景 | 策略 |
|---------|------|
| 捆绑销售 | 将高置信度组合打包销售 |
| 交叉推荐 | 购买A的用户推荐B |
| 货架陈列 | 关联商品就近摆放 |
| 促销设计 | 买A送B，提升连带率 |

**触发条件**: 用户提到"价格弹性"、"价格敏感度"、"定价优化"、"elasticity"等

**核心公式**:
```
价格弹性系数 = (需求量变化%) / (价格变化%)
             = (ΔQ/Q) / (ΔP/P)

弹性分类:

- |E| = 1: 单位弹性，收入最大化点

```

**定价策略**:

| 低弹性 | 价格不敏感 | 溢价空间，提升毛利 |

## 业务场景工作流示例

### 场景1: 月度业绩复盘

**业务背景**:
运营经理需要复盘上月业绩，理解销售额变动原因。

**工作流**:
```
1. 数据清洗
   输入: 原始销售数据(订单、SKU、金额、时间)
   输出: 清洗后的标准化数据

2. 趋势分析
   输入: 清洗后的销售数据
   分析: MoM增长率、YoY增长率、MAT趋势
   输出: 趋势分析报告

3. 归因分析
   输入: 销售额、访客数、转化率、客单价
   分析: 各因素对销售额变动的贡献度
   输出: 贡献度分解报告

4. 产品分析
   输入: SKU级别销售数据
   分析: ABC分类、盈利矩阵
   输出: 产品组合优化建议

5. 可视化报告
   输出: 综合复盘PPT/Excel仪表板
```

**使用Skills**:
- `cbec-data-cleaning`
- `cbec-yoy-mom-analysis`
- `cbec-contribution-calculation`
- `cbec-profitability-matrix`

## 附录：算法与模型参考

### A. 归因分析算法详解

#### A.1 三大类归因算法

**第一类: 单点归因**
- 首次触点归因: 100%归给第一个渠道
- 末次触点归因: 100%归给最后一个渠道

**第二类: 多点归因**
- 线性归因: 平均分配
- 时间衰减归因: 越近权重越高
- 位置归因(U型): 首末高，中间低

**第三类: 算法归因**
- Shapley值: 博弈论公平分配
- Markov链: 计算移除效应
- 机器学习: 基于特征的重要性

#### A.2 贡献度计算方法

**加法型指标**:
```
总销售额 = 品类A销售 + 品类B销售 + 品类C销售
各品类贡献 = 该品类销售额 / 总销售额
```

**乘法型指标**:
```
销售额 = 访客 × 转化率 × 客单价
使用链式替代法或Shapley值
```

### B. 预测模型参考

#### B.1 时间序列模型选择树

```
数据是否有明显趋势?
├─ 否 → 数据是否有季节性?
│   ├─ 否 → 使用移动平均
│   └─ 是 → 使用季节性ARIMA
└─ 是 → 趋势是否线性?
    ├─ 否 → 使用Prophet或LSTM
    └─ 是 → 使用ARIMA或线性回归
```

#### B.2 LSTM参数调优指南

| 问题 | 解决方案 |
|------|---------|
| 过拟合 | 增加Dropout、减少LSTM单元、早停 |
| 欠拟合 | 增加LSTM层、增加训练轮数 |
| 波动大 | 增加时间步长、添加更多历史特征 |
| 预测滞后 | 减少时间步长、调整学习率 |

### C. 财务分析公式汇总

#### C.1 杜邦分析完整公式

```
ROE = 净利润率 × 总资产周转率 × 权益乘数

展开:
ROE = (净利润/营业收入) × (营业收入/总资产) × (总资产/股东权益)

五因素扩展:
ROE = (净利润/税前利润) × (税前利润/EBIT) × (EBIT/营业收入) × (营业收入/总资产) × (总资产/股东权益)
    = 税收负担 × 利息负担 × 营业利润率 × 资产周转率 × 权益乘数
```

#### C.2 关键财务比率

| 类别 | 比率 | 公式 | 意义 |
|------|------|------|------|
| 盈利能力 | 毛利率 | (收入-成本)/收入 | 产品盈利空间 |
| 盈利能力 | 净利率 | 净利润/收入 | 最终盈利水平 |
| 运营效率 | 库存周转 | 销售成本/平均库存 | 库存管理效率 |
| 运营效率 | 应收周转 | 收入/平均应收 | 回款速度 |
| 偿债能力 | 流动比率 | 流动资产/流动负债 | 短期偿债能力 |
| 偿债能力 | 资产负债率 | 总负债/总资产 | 财务风险 |
