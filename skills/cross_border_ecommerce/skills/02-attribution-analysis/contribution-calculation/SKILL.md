# 跨境电商指标归因与贡献度计算 Skill

## 概述

在跨境电商运营中，核心指标（销售额、毛利、转化率等）的变动往往由多个因素共同驱动。本Skill提供系统化的归因分析方法，帮助用户：
- **量化各因素贡献**: 明确知道每个因素对指标变动的具体影响
- **定位关键驱动**: 识别对指标影响最大的核心因素
- **支持决策优化**: 基于归因结果制定针对性改进策略

### 支持的指标类型

| 指标类型 | 示例 | 分析方法 |
|---------|------|---------|
| 加法型 | 总销售额 = 各品类销售之和 | 直接占比法 |
| 乘法型 | 销售额 = 访客 × 转化 × 客单价 | 链式替代法、Shapley值 |
| 复合型 | 毛利率 = (收入-成本)/收入 | 差异分解法 |

## 核心功能

### 1. 加法型指标归因

**适用场景**: 总指标由多个部分相加构成

**计算公式**:
```
总指标 = 部分A + 部分B + 部分C + ...

某部分贡献度 = 该部分值 / 总指标值
某部分贡献变动 = 该部分变动值 / 总指标变动值
```

**示例**:
```
总销售额 = 美国站销售 + 欧洲站销售 + 日本站销售

美国站贡献度 = 美国站销售 / 总销售额
```

### 2. 乘法型指标归因

**适用场景**: 总指标由多个因素相乘构成

**方法一: 链式替代法**
```
指标 = A × B × C

基期: V₀ = A₀ × B₀ × C₀
本期: V₁ = A₁ × B₁ × C₁

A因素贡献 = (A₁ - A₀) × B₀ × C₀
B因素贡献 = A₁ × (B₁ - B₀) × C₀
C因素贡献 = A₁ × B₁ × (C₁ - C₀)

验证: 总变动 = A贡献 + B贡献 + C贡献
```

**方法二: Shapley值法（推荐）**
```
基于博弈论，公平分配各因素贡献

φᵢ = Σ [|S|!(n-|S|-1)!/n!] × [v(S∪{i}) - v(S)]

其中:
- S: 不包含i的所有子集
- v(S): 子集S的协同价值
- n: 总因素数

优点: 考虑因素间交互作用，分配更公平
```

### 3. 差异归因分析

**预算vs实际差异**:
```
总差异 = 实际值 - 预算值

价格差异 = (实际价格 - 预算价格) × 实际数量
数量差异 = (实际数量 - 预算数量) × 预算价格
组合差异 = 产品结构变化带来的影响
```

### 4. 多维度归因

支持按多个维度拆解:
- **时间维度**: 同比、环比、MAT
- **空间维度**: 国家、地区、渠道
- **产品维度**: 品类、SKU、品牌
- **客户维度**: 新老客、客户等级

## 使用方法

### 输入数据要求

**基础数据格式**:
```csv
period,metric_a,metric_b,metric_c,target_value
2024-01,100,0.05,50,250
2024-02,120,0.06,52,374.4
```

**字段说明**:
| 字段 | 说明 | 示例 |
|------|------|------|
| period | 时间周期 | 2024-01, Q1, Week1 |
| metric_* | 分解因素 | 访客数、转化率、客单价 |
| target_value | 目标指标值 | 销售额、毛利额 |

### 分析流程

```
步骤1: 数据加载与理解
- 读取指标数据
- 识别指标类型（加法/乘法/复合）
- 确定分析维度

步骤2: 选择归因方法
- 加法型 → 直接占比法
- 乘法型 → 链式替代法或Shapley值
- 差异分析 → 因素分解法

步骤3: 计算贡献度
- 按选定方法计算各因素贡献
- 计算贡献占比
- 排序识别关键因素

步骤4: 敏感性分析
- 分析各因素敏感度
- 模拟不同情景

步骤5: 可视化与报告
- 生成瀑布图、帕累托图
- 输出归因分析报告
```

## 算法详解

### 链式替代法完整示例

**场景**: 分析销售额变动原因
```
销售额 = 访客数 × 转化率 × 客单价

基期(2024-01):
- 访客数 = 10,000
- 转化率 = 5%
- 客单价 = $50
- 销售额 = 10,000 × 0.05 × 50 = $25,000

本期(2024-02):
- 访客数 = 12,000 (+20%)
- 转化率 = 6% (+20%)
- 客单价 = $52 (+4%)
- 销售额 = 12,000 × 0.06 × 52 = $37,440 (+49.8%)

总变动 = $37,440 - $25,000 = $12,440
```

**因素贡献计算**:
```
访客数贡献 = (12,000 - 10,000) × 0.05 × 50 = $5,000 (40.2%)
转化率贡献 = 12,000 × (0.06 - 0.05) × 50 = $6,000 (48.2%)
客单价贡献 = 12,000 × 0.06 × (52 - 50) = $1,440 (11.6%)

验证: $5,000 + $6,000 + $1,440 = $12,440 ✓
```

### Shapley值计算示例

**场景**: 三个因素A、B、C，计算各因素对销售额的贡献

**步骤1**: 列出所有可能的排列组合（3! = 6种）
```
1. A→B→C
2. A→C→B
3. B→A→C
4. B→C→A
5. C→A→B
6. C→B→A
```

**步骤2**: 计算每种排列下各因素的边际贡献
```
以排列1 (A→B→C) 为例:
- A的边际贡献 = v({A}) - v({}) 
- B的边际贡献 = v({A,B}) - v({A})
- C的边际贡献 = v({A,B,C}) - v({A,B})
```

**步骤3**: 计算Shapley值（各排列边际贡献的平均）
```
φ_A = (A在所有排列中的边际贡献之和) / 6
φ_B = (B在所有排列中的边际贡献之和) / 6
φ_C = (C在所有排列中的边际贡献之和) / 6
```

**优点**: 考虑因素间交互，避免链式替代法的顺序偏差

## 示例场景

### 示例1: 销售额下降归因

**问题**: 本月销售额下降10%，需要定位原因

**数据**:
| 指标 | 上月 | 本月 | 变动 |
|------|------|------|------|
| 销售额 | $100,000 | $90,000 | -10% |
| 访客数 | 50,000 | 45,000 | -10% |
| 转化率 | 2% | 1.8% | -10% |
| 客单价 | $100 | $111 | +11% |

**归因分析**:
```
访客数贡献 = (45,000 - 50,000) × 0.02 × 100 = -$10,000 (-100%)
转化率贡献 = 45,000 × (0.018 - 0.02) × 100 = -$9,000 (-90%)
客单价贡献 = 45,000 × 0.018 × (111 - 100) = +$8,910 (+89%)

总变动 = -$10,000 - $9,000 + $8,910 = -$10,090 ≈ -$10,000
```

**结论**:
1. 访客数下降是主要原因（贡献-100%）
2. 转化率下降加剧了问题（贡献-90%）
3. 客单价提升部分抵消了负面影响（+89%）

**建议**:
- 优先解决流量问题（广告投放、SEO优化）
- 其次优化转化率（页面体验、促销策略）
- 保持客单价提升势头

### 示例2: 多品类贡献分析

**场景**: 分析各品类对总毛利的贡献

**数据**:
| 品类 | 本月毛利 | 上月毛利 | 变动 |
|------|----------|----------|------|
| 电子产品 | $50,000 | $40,000 | +$10,000 |
| 服装 | $30,000 | $35,000 | -$5,000 |
| 家居 | $20,000 | $15,000 | +$5,000 |
| **合计** | **$100,000** | **$90,000** | **+$10,000** |

**贡献度计算**:
```
电子产品贡献度 = +$10,000 / +$10,000 = 100%
服装贡献度 = -$5,000 / +$10,000 = -50%
家居贡献度 = +$5,000 / +$10,000 = 50%

验证: 100% - 50% + 50% = 100% ✓
```

**结论**:
- 电子产品是主要增长驱动（100%贡献）
- 服装品类拖累整体表现（-50%贡献）
- 家居品类正向贡献（50%）

**建议**:
- 加大电子产品投入
- 分析服装品类下滑原因
- 复制家居品类成功经验

### 示例3: 预算差异分析

**场景**: 分析实际销售额与预算的差异

**数据**:
| 指标 | 预算 | 实际 | 差异 |
|------|------|------|------|
| 销售额 | $500,000 | $450,000 | -$50,000 |
| 销售数量 | 10,000 | 9,000 | -1,000 |
| 单价 | $50 | $50 | $0 |

**差异分解**:
```
数量差异 = (9,000 - 10,000) × $50 = -$50,000
价格差异 = 9,000 × ($50 - $50) = $0

总差异 = -$50,000 + $0 = -$50,000
```

**结论**: 销售未达标完全由销量不足导致，价格执行符合预算。

## 注意事项

### 方法选择

| 场景 | 推荐方法 | 原因 |
|------|---------|------|
| 简单乘法关系 | 链式替代法 | 计算简单，易于理解 |
| 复杂交互关系 | Shapley值 | 公平分配，考虑交互 |
| 预算差异 | 因素分解法 | 符合财务分析习惯 |
| 快速估算 | 直接占比法 | 简单直观 |

### 常见问题

**Q: 链式替代法的顺序会影响结果吗？**
A: 会。不同替代顺序可能产生 slightly different 结果。建议按业务逻辑确定顺序（如先量后价），或使用Shapley值消除顺序影响。

**Q: 贡献度之和为什么不等于100%？**
A: 可能是:
1. 计算误差（四舍五入）
2. 存在交互效应
3. 数据问题（需检查）

**Q: 如何处理负值情况？**
A: 当指标为负时，贡献度解释需谨慎。建议:
1. 使用绝对值计算占比
2. 关注变动方向而非绝对贡献
3. 结合业务场景解读

### 最佳实践

1. **明确分析目标**: 先确定要解决什么问题
2. **选择合适方法**: 根据指标类型和业务场景选择
3. **验证计算结果**: 确保贡献度之和等于总变动
4. **结合业务解读**: 数字背后要有业务逻辑支撑
5. **转化为行动**: 归因结果要指向具体改进措施
