# 母婴跨境电商需求预测 Skill

## 概述

需求预测是母婴跨境电商供应链管理的核心环节。母婴产品具有独特的需求模式:
- **生命周期驱动**: 宝宝成长阶段决定需求品类
- **季节性影响**: 疾病流行、季节变化影响需求
- **节日集中**: 圣诞节、儿童节等节日需求峰值明显
- **安全敏感**: 断货会导致客户流失和信任损失
- **保质期约束**: 奶粉、辅食等产品需考虑效期管理

准确的预测可以帮助企业:
- **降低库存成本**: 避免过度库存和效期损失
- **减少缺货损失**: 确保热销产品不断货(母婴客户断货损失大)
- **优化采购计划**: 提前安排生产和物流
- **提升客户满意度**: 保证产品可得性,维持信任
- **减少效期损失**: 合理备货,避免临期产品

本Skill整合多种预测算法，充分考虑母婴行业特性，根据数据特征自动选择最优模型，提供专业级的需求预测能力。

## 核心功能

### 1. 多算法预测引擎

**支持的算法**:

| 算法 | 适用场景 | 优势 | 局限 | 母婴适用性 |
|------|---------|------|------|-----------|
| **ARIMA** | 稳定时间序列 | 经典可靠，解释性强 | 需平稳数据，难处理复杂季节性 | 适合高频刚需品类(纸尿裤、奶粉) |
| **Prophet** | 含节假日数据 | 自动处理节假日，易用 | 对异常值敏感 | **最适合母婴**(处理节日+季节性疾病) |
| **LSTM** | 复杂非线性模式 | 捕捉长期依赖，精度高 | 需大量数据，训练时间长 | 适合复杂模式(订阅客户需求) |
| **XGBoost** | 多变量预测 | 支持多特征，可解释 | 需特征工程 | 适合多因素影响(价格+促销+季节) |
| **集成模型** | 通用场景 | 综合各算法优势，稳定 | 计算复杂度高 | **推荐**(综合母婴复杂因素) |

### 2. 母婴专属预测因子

#### 2.1 宝宝成长周期因子
```
月龄段需求特征:
- 0-3个月: 奶粉、NB号纸尿裤、奶瓶 (高频刚需)
- 4-6个月: S号纸尿裤、辅食引入准备
- 7-12个月: M/L号纸尿裤、辅食、爬行垫
- 13-24个月: XL号纸尿裤、早教玩具、童装
- 25个月+: 教育玩具、图书、安全座椅

预测调整:
需求(t) = 基础预测 × 月龄调整系数 × 季节系数 × 节日系数

月龄调整系数:
- 0-6个月: 1.2 (高频刚需期)
- 7-12个月: 1.0 (稳定期)
- 13-24个月: 0.9 (需求多样化)
- 25个月+: 0.7 (可选品类增加)
```

#### 2.2 季节性疾病影响因子
```
疾病流行对需求的影响:
- 流感季(11月-次年2月): 维生素、消毒用品 +30-50%
- 过敏季(春季): 抗过敏产品 +20-30%
- 肠胃疾病季(夏季): 益生菌、电解质水 +15-25%
- 手足口病季(5-7月): 消毒用品、体温计 +25-40%

预测调整:
需求(t) = 基础预测 × (1 + 疾病影响系数)
```

#### 2.3 母婴节日日历
```
关键节日及影响品类:
- 春节(1-2月): 礼盒装、新衣服装 +50-100%
- 儿童节(6月1日): 玩具、图书 +40-60%
- 暑假(7-8月): 戏水玩具、防晒用品 +30-50%
- 万圣节(10月31日): 服装、装饰品 +100-200%
- 圣诞节(12月25日): 玩具、礼盒 +150-300%
- 黑五/网一(11月底): 全品类 +80-150%

预测策略:
- 节日前2-4周: 开始备货
- 节日前1周: 需求峰值
- 节日后1-2周: 需求回落(透支效应)
```

### 3. 自动算法选择(母婴优化)

根据数据特征和母婴品类特性智能推荐算法:
```
高频刚需品类(纸尿裤/奶粉)?
  └─ 数据量 < 100条? → ARIMA (稳定预测)
  └─ 数据量 >= 100条? → Prophet + 节假日效应

节日礼品品类(玩具/礼盒)?
  └─ 有明显季节性? → Prophet (自动季节性分解)
  └─ 多节日影响? → 集成模型 (综合处理)

季节性品类(服装)?
  └─ 有2年+数据? → Prophet + 季节性调整
  └─ 数据不足? → ARIMA + 人工调整

订阅制产品?
  └─ 稳定订阅客户? → 简单移动平均
  └─ 新订阅增长? → LSTM (捕捉增长趋势)

不确定? → 集成模型(推荐)
```

**支持的算法**:

| 算法 | 适用场景 | 优势 | 局限 |
|------|---------|------|------|
| **ARIMA** | 稳定时间序列 | 经典可靠，解释性强 | 需平稳数据，难处理复杂季节性 |
| **Prophet** | 含节假日数据 | 自动处理节假日，易用 | 对异常值敏感 |
| **LSTM** | 复杂非线性模式 | 捕捉长期依赖，精度高 | 需大量数据，训练时间长 |
| **XGBoost** | 多变量预测 | 支持多特征，可解释 | 需特征工程 |
| **集成模型** | 通用场景 | 综合各算法优势，稳定 | 计算复杂度高 |

### 2. 自动算法选择

根据数据特征智能推荐算法:
```
数据量 < 100条? → ARIMA
含节假日信息? → Prophet
数据量 > 1000条且模式复杂? → LSTM
多特征可用? → XGBoost
不确定? → 集成模型(推荐)
```

### 3. 预测评估与验证

**评估指标**:
```
MAPE (平均绝对百分比误差) = (1/n) × Σ|实际-预测|/实际 × 100%
RMSE (均方根误差) = √[Σ(实际-预测)²/n]
MAE (平均绝对误差) = Σ|实际-预测|/n
SMAPE (对称MAPE) = (1/n) × Σ|实际-预测|/((实际+预测)/2) × 100%
```

**交叉验证**:
- 时间序列交叉验证
- Walk-forward验证
- 多步预测验证

### 4. 置信区间与不确定性

提供预测的不确定性范围:
```
点预测: 100件
95%置信区间: [85, 115]件
意味着: 有95%概率实际销量在85-115件之间
```

### 5. 库存优化建议

基于预测结果生成库存策略:
```
建议安全库存 = 预测误差标准差 × 服务水平系数
建议补货点 = 预测需求 + 安全库存
建议补货量 = 经济订货量(EOQ)模型
```


```
- 统计描述性指标

- 划分训练集/测试集
- 训练多个候选模型
- 交叉验证评估
- 选择最佳模型

步骤5: 生成预测
- 未来N期预测
- 计算置信区间
- 预测结果可视化

步骤6: 输出报告
- 预测结果表
- 模型评估报告
- 库存优化建议
```

## 算法详解

### ARIMA模型

**模型结构**: ARIMA(p,d,q)
- p: 自回归阶数
- d: 差分阶数（使序列平稳）
- q: 移动平均阶数

**参数选择**:
```python
# 自动选择最优参数
from pmdarima import auto_arima

model = auto_arima(
    data,
    seasonal=True,  # 季节性
    m=7,           # 季节周期（7天）
    start_p=0, max_p=5,
    start_q=0, max_q=5,
    d=None,        # 自动确定差分阶数
    trace=True,
    error_action='ignore',
    suppress_warnings=True
)
```

**适用场景**: 数据量较小(<500条)、模式相对稳定的SKU

### Prophet模型

**模型结构**:
```
y(t) = g(t) + s(t) + h(t) + εt

其中:
- g(t): 趋势项（线性或逻辑增长）
- s(t): 季节性项（傅里叶级数）
- h(t): 节假日效应
- εt: 误差项
```

**节假日设置**:
```python
from fbprophet import Prophet

# 定义节假日
holidays = pd.DataFrame({n    'holiday': 'black_friday',
    'ds': pd.to_datetime(['2024-11-29', '2025-11-28']),
    'lower_window': -1,
    'upper_window': 1,
})

model = Prophet(holidays=holidays)
model.fit(df)
```

**适用场景**: 有明显节假日效应、需要解释性的场景

### LSTM模型

**网络结构**:
```
输入层 → LSTM层(128单元) → Dropout(0.2) → 
LSTM层(64单元) → Dropout(0.2) → 
全连接层(32单元) → 输出层(1单元)
```

**关键超参数**:
| 参数 | 说明 | 建议值 |
|------|------|--------|
| timesteps | 历史时间步 | 30-90天 |
| units | LSTM单元数 | 64-256 |
| dropout | 丢弃率 | 0.2-0.5 |
| learning_rate | 学习率 | 0.001-0.01 |
| batch_size | 批次大小 | 32-128 |
| epochs | 训练轮数 | 50-200 |

**适用场景**: 数据量大(>1000条)、模式复杂、长期预测

### 集成模型

**集成策略**:
```python
# 加权平均集成
weights = {'arima': 0.2, 'prophet': 0.3, 'lstm': 0.5}
ensemble_pred = (
    weights['arima'] * arima_pred +
    weights['prophet'] * prophet_pred +
    weights['lstm'] * lstm_pred
)

# 权重基于历史MAPE表现
# MAPE越低，权重越高
```

**优势**:
- 综合各算法优点
- 降低单一模型风险
- 提高预测稳定性

### 示例4: 母婴纸尿裤需求预测(母婴专属场景)

- 品类: 高频刚需
- 客户特征: 60%为订阅客户
- 季节性: 轻微(夏季略低)

**预测策略**:

1. **多因素模型**
```python

    weekly_seasonality=True,
    daily_seasonality=False

```

2. **预测结果**

```markdown
- 预测期间: 2024-03-01 至 2024-05-31 (3个月)
- 最佳模型: Prophet + 母婴因子 (MAPE: 6.2%)
- 置信水平: 95%

## 月度预测

| 月份 | 点预测 | 下限(95%) | 上限(95%) | 关键影响因子 |
|------|--------|-----------|-----------|-------------|
```

## 注意事项

### 数据质量
- **缺失值**: 避免大量缺失，特别是连续缺失
- **异常值**: 异常大促或缺货期需标记处理
- **数据长度**: 建议至少2个完整周期数据

### 模型局限
- **黑天鹅事件**: 无法预测突发事件（疫情、政策变化）
- **新品预测**: 历史数据少，预测不确定性高
- **长尾SKU**: 销量少且不稳定，预测难度大

### 最佳实践
1. **多模型对比**: 不要依赖单一模型
2. **滚动预测**: 定期更新模型和预测
3. **人工校验**: 结合业务经验调整
4. **监控反馈**: 追踪预测准确性，持续优化
