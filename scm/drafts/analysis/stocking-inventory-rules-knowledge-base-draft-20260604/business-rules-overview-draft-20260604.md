# 备货库存业务规则总览

## 业务目标

备货库存规则的目标是支持备货计划使用统一库存视图，覆盖中仓、FBA、Walmart、TikTok、Shopify、三方仓等渠道。

产品能力目标：

- 展示数据源库存。
- 展示计划业务库存。
- 支持筛选、排序、导出。
- 支持同步时间展示。
- 保证映射数据准确性和时效性。

## 版本规划

| 版本 | 目标 | 规则含义 |
|---|---|---|
| 0.5 | 保证映射数据准确性 | 先解决 SKU/MSKU/FNSKU/GTIN/ItemID/仓库归属映射 |
| 1.0 | 完成数据源 + 业务库存展示 | 对接积加和平台数据，展示业务库存表 |
| 2.0 | 完成库存中心数据源切换 | 后续切换为平台对接或库存中心统一数据源 |

## 库存规则分层

| 层级 | 含义 | 示例 |
|---|---|---|
| 数据源库存 | 直接来自积加、平台接口、SRM、自研 ERP 的库存或单据数量 | 积加可用量、FBA 本地可售、物流跟踪数量 |
| 业务派生库存 | 由数据源字段组合后的业务字段 | 在库库存、在库良品库存 |
| 计划业务库存 | 支持备货计划的跨来源库存口径 | 计划库存、采购未交库存、在途库存 |
| 展示与导出口径 | 页面搜索、筛选、排序、导出规则 | 计划业务 FBA 库存导出 |

## 业务库存核心字段

| 字段 | 统一含义 | 是否源字段 |
|---|---|---|
| 计划库存 | 未来将进入目标渠道或仓库的计划性库存 | 否，按平台规则派生 |
| 采购未交库存 | 已确认采购但未完成交付和收货的库存 | 否，SRM/采购/送货单派生 |
| 在途库存 | 已发出但未完成入库或平台接收的库存 | 否，按货件、调拨、物流跟踪派生 |
| 可用库存 | 可用于销售或备货判断的库存 | 可能是源字段或平台派生字段 |
| 预占库存 | 已被订单、买家订单、运营中心或仓库占用的库存 | 可能是源字段或平台派生字段 |
| 冻结库存 | 业务不可直接使用但仍属良品库存范围的库存 | 多平台依赖默认 0 或平台字段 |
| 在库库存 | 可用 + 预占 + 冻结 + 不良品 | 派生字段 |
| 在库良品库存 | 可用 + 预占 + 冻结 | 派生字段 |
| 不良品库存 | 次品、不可售、拒收损坏等不可正常销售库存 | 源字段或平台派生 |

## 主业务流

```text
SRM 需求池/采购订单/送货单
  -> 计划库存/采购未交库存
调拨单/FBA 货件/物流跟踪表
  -> 计划库存/在途库存
积加产品库存/平台仓库存/三方仓库存
  -> 可用/预占/冻结/不良品
平台接口库存/FBT 库存
  -> 平台可用/预占/不良品
仓库归属/销售映射
  -> 渠道、国家、业务主键归并
业务库存表
  -> 展示、筛选、排序、导出、备货计划引用
```

## 关键风险

| 风险 | 表现 | 治理要求 |
|---|---|---|
| 同名字段多公式 | FBA、Walmart、TikTok 的可用库存公式不同 | 按平台建立规则 |
| 一对多映射放大 | GTIN 对多个 ItemID，供应链 SKU 对多个 MSKU | 记录映射倍数和未匹配明细 |
| 默认 0 被误解 | Walmart 预占、冻结默认为 0 | 标记 `business-default-zero` |
| 数据源混用 | 积加、自研 ERP、平台接口混合 | 必须带 `source_system` |
| 状态过滤遗漏 | 调拨、货件、物流跟踪状态直接影响计划和在途 | 状态必须入维表 |
