# 跨境电商品类管理 Skill

## 概述

品类管理是跨境电商产品运营的核心工作。通过科学的产品分类和分层管理，企业可以：
- **聚焦核心产品**: 将资源集中在高价值产品上
- **优化库存配置**: 不同品类采用不同的库存策略
- **提升整体盈利**: 淘汰低效产品，放大优势产品
- **指导选品决策**: 基于数据分析优化产品组合

本Skill提供ABC分析、帕累托分析、品类矩阵等经典工具，帮助用户系统化进行品类管理。

## 核心功能

### 1. ABC分类分析

**帕累托原理**: 20%的产品贡献80%的价值

**分类标准**:
| 类别 | 产品占比 | 销售额贡献 | 管理策略 |
|------|----------|------------|----------|
| **A类(头部)** | 10-20% | 70-80% | 重点管理，精细运营 |
| **B类(腰部)** | 20-30% | 15-20% | 常规管理，适度关注 |
| **C类(长尾)** | 50-70% | 5-10% | 简化管理，自动化 |

**多维度ABC分析**:
- 按销售额ABC
- 按利润额ABC
- 按销量ABC
- 综合评分ABC

### 2. 品类盈利矩阵

**波士顿矩阵变体**:
```
          高利润率
              │
    明星产品  │  问题产品
    (重点投资)│  (优化或淘汰)
              │
低销量────────┼────────高销量
              │
    观察产品  │  现金牛产品
    (考虑淘汰)│  (维持收割)
              │
          低利润率
```

**四象限策略**:
| 象限 | 特征 | 策略 |
|------|------|------|
| 明星 | 高销量高利润 | 加大投入，扩大份额 |
| 现金牛 | 高销量低利润 | 维持现状，稳定收割 |
| 问题 | 低销量高利润 | 分析原因，提升销量或淘汰 |
| 观察 | 低销量低利润 | 考虑淘汰 |

### 3. 产品生命周期分析

识别产品所处阶段:
- **引入期**: 新上架，销量低但增长快
- **成长期**: 销量快速增长
- **成熟期**: 销量稳定，贡献主要收入
- **衰退期**: 销量下滑，考虑淘汰

### 4. 品类组合优化

基于分析结果提供优化建议:
- 加大投入品类
- 维持现状品类
- 优化改进品类
- 考虑淘汰品类

## 算法说明

### ABC分类算法

```python
def abc_analysis(data, value_column, a_threshold=0.8, b_threshold=0.95):
    """
    ABC分类算法
    
    参数:
    - data: DataFrame, 包含产品数据
    - value_column: 价值字段名(销售额/利润)
    - a_threshold: A类累计贡献阈值(默认80%)
    - b_threshold: B类累计贡献阈值(默认95%)
    """
    # 按价值降序排序
    sorted_data = data.sort_values(value_column, ascending=False)
    
    # 计算累计贡献
    total_value = sorted_data[value_column].sum()
    sorted_data['cumulative'] = sorted_data[value_column].cumsum()
    sorted_data['cumulative_pct'] = sorted_data['cumulative'] / total_value
    
    # 分类
    sorted_data['abc_class'] = 'C'
    sorted_data.loc[sorted_data['cumulative_pct'] <= a_threshold, 'abc_class'] = 'A'
    sorted_data.loc[
        (sorted_data['cumulative_pct'] > a_threshold) & 
        (sorted_data['cumulative_pct'] <= b_threshold), 
        'abc_class'
    ] = 'B'
    
    return sorted_data
```

### 综合评分模型

结合多维度计算产品综合得分:
```
综合得分 = w1×销售额得分 + w2×利润得分 + w3×周转得分 + w4×增长得分

权重建议:
- w1 (销售额): 0.35
- w2 (利润): 0.35
- w3 (周转): 0.20
- w4 (增长): 0.10

得分标准化: 采用Min-Max标准化到0-100分
```

## 示例场景

### 示例1: 电子产品线ABC分析

**数据**: 某店铺100个电子SKU的月销售数据

**分析结果**:
```
A类(20 SKU, 75%销售):
- 蓝牙耳机: $45,000
- 智能手表: $38,000

...

发现:
- 前3个SKU贡献35%销售额
- 建议重点维护这3个SKU的库存和排名
```

**行动**:
1. 为A类SKU设置独立库存预警
2. 每日监控A类SKU的销售排名
3. 与A类SKU供应商签订优先供货协议

### 示例2: 全品类盈利矩阵

**数据**: 全店500个SKU的销售和利润数据

**矩阵分布**:
```
明星产品(15 SKU): 高销高利
- 策略: 加大投入，扩大优势

现金牛产品(30 SKU): 高销低利
- 策略: 优化成本，提升利润

问题产品(25 SKU): 低销高利
- 策略: 分析原因，提升销量

**优化方案**:
- 淘汰80个观察产品
- 对25个问题产品进行促销测试
- 为15个明星产品开发变体

### 示例3: 库存优化决策

**场景**: 库存资金紧张，需要优化库存结构

**分析**:
```
当前库存分布:
- A类产品: 占库存资金60% (合理)
- B类产品: 占库存资金30% (偏高)
- C类产品: 占库存资金10% (应降低)

优化目标:
- A类: 维持60%
- B类: 降至25%
- C类: 降至5% (按需采购)

**行动**:
1. C类产品: 停止主动补货，清仓处理
2. B类产品: 降低安全库存至2周
3. A类产品: 确保4周安全库存

**预期效果**:
- 释放库存资金: 20%
- 降低滞销风险
- 提升库存周转率

## 注意事项

### 分类标准调整
- 不同行业ABC阈值可能不同
- 可根据实际情况调整(如A类70%或85%)
- 建议定期review分类结果

### 动态管理
- ABC分类不是一成不变的
- 建议每月更新分类
- 关注产品类别变化趋势

### 多维度平衡
- 不仅看销售额，还要看利润
- 考虑库存周转效率
- 关注产品增长潜力
