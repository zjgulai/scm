# KPI 目录说明

本目录用于沉淀个人月度 KPI，聚焦 `main_project_lute` 四大专题与交叉故事线的数据分析产出。

## 使用方式

- 每月新建一个文件，命名建议：`YYYY-MM_KPI.md`。
- 目标项需可量化、可验收，且与专题/子课题/交叉线存在明确映射。
- 一～三项权重建议总和为 100%，并在月末进行复盘更新。
