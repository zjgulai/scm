# 积加 SCM 跨模块指标血缘

## 一句话判断

积加 SCM 四模块不是并列关系，而是一个以 `库存需求` 为起点、以 `库存状态和成本归集` 为终点的单据链路。计划模块负责计算需求，采购模块负责供应商履约，仓库模块负责实物状态，物流模块负责节点移动和成本。

## 主链路

```text
产品/MSKU/仓库/站点/供应商/物流方式主数据
  -> 销售订单和目标销量
  -> 智能补货/红绿线/连续补货建议
  -> 采购计划/发货计划/装配计划
  -> 分单任务
  -> 采购订单/预装需求/委外订单
  -> 交货单
  -> 发货单/头程单/FBA货件/调拨单
  -> 入库单/平台仓接收/三方仓入库
  -> 产品库存/仓库库存/批次库存/库存流水
  -> 物流费用/供应商对账/库存成本
```

## 关键单据血缘

| 上游 | 下游 | 关联键 | 业务含义 |
|---|---|---|---|
| 智能补货建议 | 采购计划 | SKU/MSKU、目的仓、物流方式、推荐采购量 | 把库存缺口转成采购需求 |
| 智能补货建议 | 发货计划 | SKU/MSKU、发货仓、目的仓、物流方式、建议发货量 | 把库存缺口转成物流排期 |
| 采购计划 | 分单任务 | 计划单号、SKU、计划量 | 把计划量分配给供应商或装配仓 |
| 分单任务 | 采购订单 | SKU、供应商、分单数量 | 非组合品或子件采购 |
| 分单任务 | 预装需求 | 组合品、装配仓、装配量 | 本地仓组装需求 |
| 分单任务 | 委外订单 | 组合品、供应商仓、装配量 | 供应商仓组装需求 |
| 采购订单 | 交货单 | PO 单号、SKU、交货量 | 供应商执行交付 |
| 交货单 | 发货单 | LN 单号、发货量、交货仓/目的仓 | 货物进入运输或到仓流程 |
| 调拨单 | 发货单 | 调拨单号、调拨量、调出仓/调入仓 | 仓间库存移动 |
| 交货单/调拨单 | 头程单 | 来源单号、目的仓 | FBA 头程桥接货件 |
| 头程单 | FBA 货件 | 货件号、申报量 | 与亚马逊后台货件计划对应 |
| 发货单 | 入库单 | 发货单号、目的仓、应收货 | 自营仓收货入库 |
| 入库单 | 库存流水 | 入库单号、SKU、良品/次品变化 | 实物库存增加 |
| 出库/调拨/调整 | 库存流水 | 来源单号、SKU、良品/次品变化 | 实物库存减少或状态调整 |
| 发货单 | 物流费用 | 发货单号、物流商、费用项 | 预估和实际费用归集 |
| 采购订单/发货单/入库单 | 供应商对账 | 供应商、单据号、数量、金额 | 供应商付款核对 |

## 核心指标血缘

### 推荐采购量

```text
销售订单/目标销量
  -> 日均消耗量
  -> 目标库存天数
  -> 目标库存量
  -> 减：良品可用量
  -> 减：国内仓可调拨量
  -> 减：海外仓可调拨量
  -> 减：已下单量
  -> 减：计划入库
  -> 减：待出运
  -> 减：海外仓在途量
  -> 减：在途量
  -> 推荐采购量
```

血缘风险：

- `目标库存天数` 受采购交期、质检时间、物流时效、入库处理天数、安全库存天数等参数影响。
- `在途量` 可能取 ERP 单据，也可能取 FBA 平台报告。
- `良品可用量`、`预占量`、`可调拨量` 必须按仓库类型和站点锁定规则拆开。

### 采购执行量

```text
采购计划计划量
  -> 分单数量
  -> 采购订单采购量
  -> 交货单交货量/实交量
  -> 发货单发货量
  -> 入库单已入库/良品/次品
```

核心监控：

- 计划分单率 = 已分单量 / 计划量。
- 采购交付率 = 实交量 / 采购量。
- 入库完成率 = 已入库量 / 应收货。
- 良品率 = 入库良品量 / 已入库量。

### 库存可用量

```text
入库单/盘点/调整/调拨/出库
  -> 库存流水
  -> 良品量/次品量变化
  -> 产品库存快照
  -> 预占量/冻结量/锁站点/共享库存
  -> 可用量/可调拨量
```

血缘风险：

- 产品库存（NEW）允许平台仓库存指标映射配置，平台仓指标不是固定公式。
- 仓库库存是仓库维度汇总，不是明细来源。
- 批次库存用于实物追溯，不应替代库存汇总表。

### 物流时效

```text
物流方式
  -> 运输时效
  -> 目标库存天数
  -> 推荐发货日期/最迟发货日期
  -> 发货单确认出运日期
  -> 预计到仓日期
  -> 实际到仓/开始收货/完成日期
```

核心监控：

- 预计时效 = 预计到仓日期 - 出运日期。
- 实际时效 = 实际到仓日期 - 实际发货日期。
- 时效偏差 = 实际时效 - 预计时效。

### 物流费用

```text
运费模板/物流渠道/物流方式
  -> 运费试算
  -> 发货单预估费用
  -> 物流商实际费用
  -> 物流费用看板
  -> 估实差异看板
  -> 头程物流对账
  -> 成本/利润
```

血缘风险：

- 跨月录入实际费用时，已销售出库部分可能仍按预估成本处理。
- 差异金额需要独立归集，不能简单覆盖历史成本。

## 指标分层

| 层级 | 指标类型 | 示例 |
|---|---|---|
| L0 源字段 | ERP 页面或导出字段 | 计划量、采购量、交货量、已入库、物流方式 |
| L1 状态指标 | 单据状态和数量状态 | 待分单量、未交量、在途量、已调入量 |
| L2 计算指标 | 公式计算结果 | 目标库存量、推荐采购量、差异率 |
| L3 运营指标 | 经营判断指标 | 缺货风险、采购准交率、库存周转、物流成本偏差 |
| L4 决策指标 | 动作建议 | 是否下单、是否调拨、是否换物流方式、是否补差异成本 |

## 建设要求

1. 所有 L2 以上指标必须记录依赖字段和参数。
2. 所有计划类指标必须记录计算日期和目标日期。
3. 所有库存类指标必须记录快照时间。
4. 所有成本类指标必须记录币种、本位币、费用项、录入时间。
5. 所有跨平台仓指标必须记录来源：ERP 单据、平台报告、三方仓接口。
