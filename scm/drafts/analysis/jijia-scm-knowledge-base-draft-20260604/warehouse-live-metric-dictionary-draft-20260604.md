# 积加 ERP 仓库模块实采指标字典

## 指标域划分

仓库指标体系应按业务动作拆分为 8 个域：

| 指标域 | 核心问题 | 代表页面 |
|---|---|---|
| 库存现存量 | 当前还有多少、在哪里、什么状态、能不能卖 | 仓库库存、产品库存、自营仓库存、平台仓库存 |
| 批次与库龄 | 库存从哪里来、多久了、成本货值是多少 | 批次库存、批次流水、库龄分析 |
| 入库与质检 | 应收、已收、送检、质检、入库是否闭环 | 入库单、质检单 |
| 库内作业 | 调拨、调整、盘点、状态变更、装配、装箱是否完成 | 调拨单、调整单、盘点单、装配单、装箱任务 |
| 出库履约 | 出库、拣货、验货、发货、包裹是否推进 | 销售出库单、拣货任务、扫码发货、包裹单 |
| 物流费用 | 预估和实际物流费用、体积重量、发运状态 | 物流下单、运输单、包裹单 |
| 三方仓协同 | 三方仓入库、出库、库存、状态同步是否一致 | 三方仓入库单、三方仓销售出库单、三方仓库存差异 |
| 稽核对账 | ERP、三方仓、FBA 报告和财务结账差异 | 三方仓库存差异、月度FBA报告差异、FBA库存对账 |

## 二次重采对指标建设的影响

2026-06-04 二次重采确认仓库模块不是无数据页面集合。当前筛选下可直接作为指标建设入口的页面包括：仓库库存、产品库存、自营仓库存、平台仓库存、批次库存、调拨单、调整单、其它出入库、销售出库单、包裹单、三方仓入库单、三方仓库存差异、仓库资料、库存批次初始化。

当前页面级总量只能作为 UI 证据，不得直接定义为公司级指标。原因是同一页面可能同时存在分页总数、状态页签数量、合计窗口和当前筛选条件，这几类口径并不总是一致。例子：

| 页面 | 现象 | 建模处理 |
|---|---|---|
| 入库单 | 页签显示 `待入库:818`，当前列表为 0 | 状态页签和列表事实拆开校验 |
| 调拨单 | 分页 2189，但页签显示调拨在途 4586 | 建立调拨状态快照，不直接用分页替代状态量 |
| 包裹单 | 分页 55937，页签 `全部` 为 1626152 | 包裹列表、状态总览、时间筛选必须分层 |
| 三方仓销售出库单 | 表格可见记录样本，但分页总数为 0 | 后续通过导出或 API 确认真实分页口径 |

## 指标口径关系

库存现存量不应只看一个字段。建议建立以下派生关系：

```text
总库存数量
= 在库量 + 在途量 + 其他页面定义纳入总库存的计划/货件/待入库数量

在库量
= 良品量 + 次品量

可用量
= 在库良品库存 - 预占量 - 出库占用数量

滞销率
= 滞销量 / 总库存数量

抽检次品率
= 抽检次品量 / 抽检量

库存差异数量
= ERP 侧对账库存量 - 外部系统侧对账库存量
```

上面的关系需要在后续导出或 API 字段拿到后校验。当前 UI 能确认指标名称和业务关联，不能确认所有字段的后端计算公式。

## 库龄口径

页面说明给出了两个关键事实：

- 库龄预警数据每日 0 时更新。
- 货值按本位币计算，汇率取系统汇率设置中当前月份官方汇率。

因此库龄指标要单独落快照表，不能和实时库存流水混算。建议库龄快照至少包含：

- `snapshot_date`
- `warehouse_id`
- `sku`
- `platform_site`
- `age_bucket`
- `quality_scope`
- `qty`
- `value_base_currency`
- `exchange_rate_month`

## 状态类指标

状态类指标要统一成标准枚举层，而不是每张表各自写中文状态：

| 状态域 | 页面示例 | 标准化方向 |
|---|---|---|
| 入库状态 | 待收货、待入库、已完成、已取消 | inbound_status |
| 质检状态 | 工厂质检、仓库质检、进行中 | qc_status, qc_type |
| 调拨状态 | 待提交、审核中、待拣货、待出库、调拨在途、已完成、已作废 | transfer_status |
| 出库状态 | 待处理、已出库、已取消 | outbound_status |
| 拣货状态 | 待拣货、拣货中、拣货完成 | picking_status |
| 物流状态 | 下单异常、下单中、已下单、已发运、已拦截 | logistics_order_status |
| 三方仓状态 | 待下单、待入库、部分入库、已完成、下单失败、出库异常 | third_party_order_status |
| 对账处理状态 | 未处理、处理中、已处理、处理失败 | reconciliation_status |

## 指标建设优先级

| 优先级 | 指标组 | 原因 |
|---|---|---|
| P0 | 可用量、预占量、在途量、良品量、次品量、总库存 | 直接影响销售可售、补货和缺货判断 |
| P0 | 库存流水、批次流水、调拨、入库、出库 | 是库存快照可解释性的根因链路 |
| P1 | 库龄段数量、库龄段货值、滞销量、滞销率 | 影响清仓、资金占用和采购节奏 |
| P1 | 入库应收、已收、送检、质检良品、抽检次品率 | 影响供应商交付质量和入库周期 |
| P1 | 拣货、出库、包裹、物流费用 | 影响履约效率和物流成本 |
| P2 | 三方仓差异、FBA 月报差异、FBA 对账差异 | 影响库存可信度和财务结账 |
| P2 | 装配、装箱、容器、库位 | 影响仓内作业效率和精细化 WMS 管理 |
