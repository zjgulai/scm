# 购物篮分析（Market Basket Analysis）

## 概述

购物篮分析基于交易明细数据，通过关联规则算法（Apriori/FP-Growth）挖掘哪些商品经常被一起购买，识别高价值商品组合和需求场景，为搭售策略和促销组合设计提供数据支撑。

## 分析框架

### 第一步：数据准备

- 以订单为单位，汇总每个订单包含的所有 SKU/品线
- 过滤低频商品（出现次数 < 阈值）和单件订单
- 转换为事务格式（每行一个订单的商品列表）

### 第二步：关联规则挖掘

- 使用 Apriori 算法计算：
  - **支持度（Support）**: P(A∩B) — 组合出现的概率
  - **置信度（Confidence）**: P(B|A) — 买了A后买B的概率
  - **提升度（Lift）**: P(B|A)/P(B) — 组合的关联强度（>1 表示正相关）

### 第三步：场景归类

- 按置信度和提升度阈值筛选有效规则
- 将规则映射到典型需求场景：
  - 互补型（如奶瓶+奶嘴）→ 功能配套
  - 补充型（如吸奶器+储奶袋）→ 使用流程
  - 替代型（纸尿裤不同尺码）→ 尺码升级

### 第四步：策略输出

- 对高提升度组合 → 建议捆绑销售/套装定价
- 对高置信度组合 → 建议交叉推荐/购物车追加
- 对低置信度高销售额商品 → 建议单独分析购买路径

## 与 category-management 的关系

- `category-management` 侧重品类维度的市场结构分析（波士顿矩阵、品类角色）
- `basket-analysis` 侧重交易维度的关联规则挖掘（哪些商品一起买）
- 两者互补：category-management 回答"卖什么"，basket-analysis 回答"怎么搭配卖"

## 适用场景

- 母婴电商：奶瓶+奶嘴、吸奶器+储奶袋、纸尿裤+湿巾
- 搭售策略设计、促销组合规划
- 新品上市时识别可关联的现有热销品

## 数据要求

- 交易明细表（订单ID + SKU + 数量）
- 最小样本量：> 500 个含多商品的订单
- 推荐最小支持度阈值：0.5%（母婴品类）
