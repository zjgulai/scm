# 专题×交叉线 字段口径说明

> 与 01_专题课题_数据需求矩阵、05_数仓表结构与主键设计 命名一致；按专题与表分组，供开发与验收时口径校验。以下仅列关键字段与口径要点，完整字段见 01/05。

## 一、专题① VOC

| 表名 | 关键字段 | 口径要点 |
|------|----------|----------|
| fact_voc_summary | voc_cnt, voc_rate, star_rating, good_rate, bad_rate, sales_qty, return_rate, channel_id, country_code, spu_id, dt_month | 粒度：渠道×国家×SPU×月；voc_rate 通常为每百单/万单 VOC 数或占比，与存量BI 一致 |
| fact_voc_trend | voc_cnt, voc_rate, voc_trend_12m, voc_source_type, mention_volume | 粒度：国家×渠道×来源类型×品类×月 |
| fact_voc_brand_summary | brand_id, voc_cnt, voc_rate, star_rating, bad_rate, tag_localized | 品牌含自有与竞品；tag_localized 为本土化主题标签 |
| ods_voc_external | platform, community_name, post_id, comment_id, topic_tags, sentiment_polarity, content_text, post_time | 帖子/评论级；topic_tags 与 fact 层 tag_l2/tag_l3 可映射 |

## 二、专题② 订单

| 表名 | 关键字段 | 口径要点 |
|------|----------|----------|
| fact_order | order_id, order_date, country_code, channel_id, gmv, cost_front_total, cost_back_*, item_qty, sku_qty, gross_margin_amt | 一行一订单；前后台成本与 momcozy-margin-analysis 口径一致（前台：促销+推广+退款；后台：生产+头程+仓配+佣金+其他） |
| fact_order_item | order_id, line_item_id, spu_id, sku_id, unit_price, item_qty, gmv_line, is_promo, is_bundle | 一行一订单行 |
| fact_return | return_id, order_id, sku_id, return_qty, return_amt, return_reason_code, is_partial_return, voc_ticket_id | 支持部分退；return_reason_code 与 dim_return_reason 关联 |
| fact_order_fulfillment | order_id, created_at, paid_at, shipped_at, in_transit_at, cleared_at, delivered_at, lead_time_* | 与 fact_order 1:1；时间均为日期时间类型 |

## 三、专题③ 渠道

| 表名 | 关键字段 | 口径要点 |
|------|----------|----------|
| fact_channel_country_month | country_code, channel_id, dt_month, gmv, sales_qty, gross_margin_amt, gross_margin_pct, lifecycle_stage, target_* | 粒度：国家×渠道×月；lifecycle_stage 为导入/成长/成熟/衰退 |
| fact_channel_traffic | country_code, channel_id, dt_month, traffic_pct_organic, traffic_pct_paid, traffic_pct_influencer, traffic_pct_promo, sessions, pv, ad_sales, gmv | 流量占比为 0～1 或百分比，总和可为 1 |
| fact_channel_health | country_code, channel_id, dt_month, gmv_yoy, margin_pct_chg, return_rate, voc_rate, target_achievement_rate | 多为计算指标，可由 channel_country_month 等衍生 |

## 四、专题④ 营销

| 表名 | 关键字段 | 口径要点 |
|------|----------|----------|
| fact_campaign_daily | campaign_id, campaign_type, country_code, channel_id, dt, ad_spend, promo_discount_amt, impressions, clicks, roas, ad_attributed_sales | 粒度：活动×国家×渠道×日 |
| fact_campaign_roi | campaign_id, country_code, channel_id, spu_id, dt, spend, attributed_sales, roas, promo_margin_roi | 归因销售额 attributed_sales 依归因模型而定 |
| fact_user_lifecycle | user_id_anon, dt_month, channel_id, country_code, lifecycle_segment, cum_gmv, ltv_segment, ltv_incr_amt | 用户级快照，按月或按事件更新 |
| fact_user_campaign | user_id_anon, campaign_id, campaign_type, event_dt | 用户参与活动明细 |

## 五、交叉线专用中间表/输出

| 产出物 | 关键字段（示例） | 口径要点 |
|--------|------------------|----------|
| 痛点/主题表（交叉线1） | theme_code, theme_name, channel_id, country_code, spu_id, priority | 由 VOC Agent 产出，订单 Agent 消费 |
| 售后/退款主题输入表（交叉线3） | order_id, return_id, return_reason_code, is_partial_return, theme_suggested, country_code, channel_id | 由订单管道产出，VOC Agent 消费；theme_suggested 与 VOC 标签体系可映射 |
| 高浓度社区清单（交叉线2） | community_name, platform, country_scope, topic_tag, priority, suggested_placement | VOC 产出，营销 Agent 消费 |
| 国家×渠道画像（交叉线4） | country_code, channel_id, lifecycle_stage, gmv, margin_pct, risk_flag | 渠道 Agent 产出，VOC/营销 Agent 消费 |

## 六、维护

- 表名字段以 01/05 为准；口径变更时同步更新本文档与数仓约定。
- 虚拟数据 mock 时按本说明生成最小字段集，便于与真实表替换时对照。
